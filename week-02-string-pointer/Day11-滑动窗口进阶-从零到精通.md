## 一、滑动窗口是什么？

### 1.1 直观理解 🎯

想象你在看一个**望远镜**：

```
🔭 望远镜的视野 = 窗口
[开始] ←—— 左端点(l)     右端点(r) ——→ [结束]
```

- 你把望远镜从左到右移动 → **滑动**
- 望远镜的视野范围可大可小 → **窗口**

**生活中的例子**：
> 你在超市排队结账，想找**总价刚好100元**的一组连续商品。
> - 你从第1个商品开始，不断往购物车里放 → 右指针扩展
> - 当总价超过100元，你从购物车前面拿出一些商品 → 左指针收缩
> - 记录每次总价刚好100元的商品数量 → 更新答案

### 1.2 为什么需要滑动窗口？

**暴力法**：枚举所有子数组，检查每个是否满足条件 → O(n²) 或 O(n³)
**滑动窗口**：左右指针各移动一次，每个元素最多进/出窗口一次 → **O(n)**

```
暴力法：n=1000时 要用1,000,000次检查
滑动窗口：n=1000时 只用2,000次操作 ❗
```

### 1.3 滑动窗口三大分类

```python
# ┌─────────────────────────────────────┐
# │  滑动窗口三大分类                     │
# ├─────────────────────────────────────┤
# │ 1. 可变窗口：大小不固定，找最优解     │
# │    例：找最长/最短满足条件的子数组     │
# ├─────────────────────────────────────┤
# │ 2. 固定窗口：大小固定，求最大/最小值  │
# │    例：大小为k的子数组最大平均值       │

# │ 3. 计数窗口：统计字符/数字频次        │
# │    例：找包含t中所有字符的最短子串     │
# └─────────────────────────────────────┘
```

## 二、常用操作大全

### 2.1 操作速查表

| 操作 | Python写法 | 示例 | 结果 | 时间复杂度 |
|------|-----------|------|------|-----------|
| 窗口求和 | `total += nums[r]` | `[1,2,3]` r=2 | total+=3 | O(1) |
| 窗口移出 | `total -= nums[l]` | `[1,2,3]` l=0 | total-=1 | O(1) |
| 窗口长度 | `r - l + 1` | l=1, r=3 | =3 | O(1) |
| 记录字符 | `seen[ch] = r` | ch='a', r=2 | {'a':2} | O(1) |
| 检查存在 | `ch in seen` | 'a' in {'a':2} | True | O(1) |
| 计数器增减 | `need[ch] -= 1` | need={'a':1} | {'a':0} | O(1) |
| 前缀和累计 | `prefix_sum += val` | 累计到第r个 | sum(nums[:r]) | O(1) |
| 哈希表查询 | `prefix.get(key, 0)` | key=s-k | 出现次数 | O(1) |

### 2.2 小白问：何时用滑动窗口？何时用前缀和？

| 题目特征 | 该用什么 |
|---------|---------|
| 找**连续子数组/子串**，条件**可动态判断**（和≥target、无重复等） | ✅ 滑动窗口（可变窗口） |
| 窗口**大小固定**，求最值 | ✅ 滑动窗口（固定窗口） |
| 找**子数组和=某个值**的**个数** | ✅ 前缀和+哈希表 |
| 找**子数组和=某个值**的**最长/最短长度** | ✅ 前缀和+哈希表 OR 滑动窗口 |
| 数组中**有负数**，滑动窗口无法收缩 | ✅ 只能用前缀和+哈希表 |

## 三、四大核心套路详解

### 🥇 套路1：可变窗口模板（最常用、最重要！）

#### 适用场景
找**最长/最短**满足某个条件的连续子数组/子串。
例如：和≥target的最短子数组、无重复字符的最长子串。

#### 核心思想
```
右指针不断扩展 → 窗口不满足条件时收缩左指针 → 每次更新答案
```

#### 可视化流程
```
数组: [2, 3, 1, 2, 4, 3], target=7

┌──────────────────────────────────────────┐
│ Step 1: r=0, total=2 < 7, 继续扩展      │
│  [2] 3  1  2  4  3                       │
│  l↑  r↑                                  │
├──────────────────────────────────────────┤
│ Step 2: r=1, total=5 < 7, 继续扩展      │
│  [2  3] 1  2  4  3                       │
│  l↑     r↑                               │
├──────────────────────────────────────────┤
│ Step 3: r=2, total=6 < 7, 继续扩展      │
│  [2  3  1] 2  4  3                       │
│  l↑        r↑                            │
├──────────────────────────────────────────┤
│ Step 4: r=3, total=8 >= 7!               │
│  收缩左指针! ans=min(∞,4)=4              │
│  [2  3  1  2] 4  3                       │
│  l↑           r↑                         │
│  移出2 → total=6 < 7 → 停止收缩          │
├──────────────────────────────────────────┤
│ Step 5: r=4, total=10 >= 7!              │
│  收缩! ans=min(4,3)=3 ✅                 │
│   2 [3  1  2  4] 3                       │
│      l↑        r↑                        │
│  移出3 → total=7, 移出1 → total=6        │
├──────────────────────────────────────────┤
│ Step 6: r=5, total=9 >= 7!               │
│  收缩! ans=min(3,2)=2 ✅                 │
│   2  3 [1  2  4  3]                      │
│          l↑     r↑                       │
│  移出1 → total=8, 移出2 → total=6        │
│  → ans=2 🎉                              │
└──────────────────────────────────────────┘
```

#### 代码模板（直接背诵！）

```python
def sliding_window_template(arr, condition_func):
    """
    可变窗口模板
    :param arr: 输入数组/字符串
    :param condition_func: 判断窗口是否满足条件的函数
    :return: 最优窗口的长度或内容
    """
    l = 0                          # 左指针（窗口左边界）
    ans = 0                        # 答案（最大/最小长度）
    # window_state = ...           # 窗口状态（和、计数、集合等）
    
    for r in range(len(arr)):      # 右指针遍历
        # ① 扩展窗口：将 arr[r] 加入窗口
        # window_state += arr[r]   # 或更新其他状态
        
        # ② 收缩窗口：当窗口不满足条件时
        while not condition_func(window_state):
            # 移除 arr[l] 从窗口
            # window_state -= arr[l]
            l += 1                  # 左指针右移
        
        # ③ 更新答案（此时窗口一定满足条件）
        ans = max(ans, r - l + 1)   # 找最长用max，找最短用min
    
    return ans
```

#### ⚠️ 关键记忆点

| 要找什么 | 收缩时机 | 更新答案位置 | 答案初始值 |
|---------|---------|-------------|-----------|
| 最长满足条件 | 不满足条件时收缩 | 内层while之后 | 0 |
| 最短满足条件 | 满足条件时收缩 | 内层while之中 | float('inf') |

### 🥈 套路2：计数窗口（含覆盖条件）

#### 适用场景
需要在窗口中统计字符/数字的频次，判断是否「全覆盖」目标。
例如：最小覆盖子串、字符串排列。

#### 核心思想
用一个计数器（字典）记录目标字符的需求量，用 `missing` 变量记录还差多少个字符未满足。

```
需求: t = "ABC"
初始: need = {'A':1, 'B':1, 'C':1}, missing = 3

扩展窗口时:
  - 遇到需要的字符 → need[ch]减1
  - 如果 need[ch] > 0 → 说明这次减少是有效的，missing减1
  - 如果 need[ch] <= 0 → 说明字符有多余

收缩窗口时:
  - 移出左指针字符
  - 如果是需要的字符 → need[ch]加1
  - 如果 need[ch] > 0 → 说明不够了，missing加1
```

#### 代码模板

```python
from collections import Counter

def count_window_template(s: str, t: str) -> str:
    """
    计数窗口模板：找包含t中所有字符的最短子串
    """
    need = Counter(t)              # 目标字符的需求字典
    missing = len(t)               # 还差多少个字符
    l = 0
    ans_start, ans_len = 0, float('inf')
    
    for r, ch in enumerate(s):
        # ① 扩展：遇到目标字符
        if ch in need:
            if need[ch] > 0:       # 这次减少是有效的
                missing -= 1
            need[ch] -= 1
        
        # ② 收缩：当所有字符都找到了
        while missing == 0:
            # 更新答案
            if r - l + 1 < ans_len:
                ans_start = l
                ans_len = r - l + 1
            
            # 移出左指针
            left_ch = s[l]
            if left_ch in need:
                need[left_ch] += 1
                if need[left_ch] > 0:  # 不够了
                    missing += 1
            l += 1
    
    return "" if ans_len == float('inf') else s[ans_start:ans_start + ans_len]
```

#### 小白问：为什么 need[ch] > 0 才减 missing？

> 答：比如 t = "AA"，need = {'A':2}。当遇到第一个'A'时，need['A']从2变1（>0），missing从2变1。遇到第二个'A'时，need['A']从1变0（>0），missing从1变0。但遇到第三个'A'时，need['A']从0变-1（=0），这时多余的'A'只是扣库存，不再影响missing——因为我们已经找够了2个'A'。

### 🥉 套路3：前缀和 + 哈希表（子数组求和利器）

#### 适用场景
找**和为k的子数组个数**、**和为0的最长子数组**等问题。
前缀和的核心思想：**两个子数组的和相减 = 中间段的和**。

#### 核心思想
```python
# 普通想法：枚举所有子数组求和 → O(n²)
# 前缀和思想：
# 前缀和 prefix[i] = sum(nums[0:i])，即前i个元素的和
# 子数组 nums[l:r] 的和 = prefix[r] - prefix[l]
# 所以：和为k的子数组 → prefix[j] - prefix[i] = k
#      → prefix[i] = prefix[j] - k 在之前出现过！

# 用哈希表记录每个前缀和出现的次数
# 遍历时查表即可 O(1) 知道有多少个前缀和 = 当前前缀和 - k
```

#### 可视化
```
数组: [1, 2, 3], 找和为3的子数组个数

┌────────────────────────────────────────────┐
│ 维护：prefix[0] = 0 (空数组的和)            │
│      哈希表: {0: 1}                         │
├────────────────────────────────────────────┤
│ i=0: nums[0]=1  → prefix=1                  │
│      查表: 1-3=-2 不存在                    │
│      记录: {0:1, 1:1}                       │
├────────────────────────────────────────────┤
│ i=1: nums[1]=2  → prefix=3                  │
│      查表: 3-3=0 存在(1次) → ans+=1         │
│      记录: {0:1, 1:1, 3:1}                  │
├────────────────────────────────────────────┤
│ i=2: nums[2]=3  → prefix=6                  │
│      查表: 6-3=3 存在(1次) → ans+=1         │
│      记录: {0:1, 1:1, 3:1, 6:1}             │
├────────────────────────────────────────────┤
│ 结果: ans=2 🎉                              │
│ 子数组: [1,2] 和 [3]                        │
└────────────────────────────────────────────┘
```

#### 代码模板

```python
from collections import defaultdict

def subarray_sum_template(nums: list[int], k: int) -> int:
    """
    前缀和+哈希表模板：统计和为k的子数组个数
    变形：和为0 → k=0
         和为k的最长长度 → 存首次出现位置而不是次数
    """
    prefix = defaultdict(int)
    prefix[0] = 1        # 重要：空数组的前缀和为0，出现1次
    cur_sum = 0
    ans = 0
    
    for val in nums:
        cur_sum += val                    # 当前前缀和
        ans += prefix[cur_sum - k]        # 关键一步！
        prefix[cur_sum] += 1              # 记录当前前缀和
    
    return ans
```

#### 常见变形

| 变形 | 改什么 | 适用场景 |
|------|--------|---------|
| 统计个数 | `prefix`存出现次数，`ans +=` | 和为k的子数组个数 |
| 最长长度 | `prefix`存首次出现索引，`ans = max(ans, i - first[sum])` | 和为k的最长长度 |
| 0/1数组 | 把0当-1处理 | 找相同数量0和1的子数组 |

### 🥉 套路4：固定窗口（滑动均值/最大值）

#### 适用场景
窗口大小固定，每滑动一步计算窗口内的值。
例如：大小为k的子数组最大平均值。

#### 代码模板

```python
def fixed_window_template(nums: list[int], k: int):
    """固定大小为k的滑动窗口"""
    # 先计算第一个窗口
    window_sum = sum(nums[:k])
    ans = window_sum
    
    for r in range(k, len(nums)):
        # 移出左边，加入右边
        window_sum += nums[r] - nums[r - k]
        ans = max(ans, window_sum)  # 或 min
    
    return ans / k if ... else ans
```

### 题1：长度最小的子数组（⭐⭐）

**题目**：给定一个正整数数组 `nums` 和一个正整数 `target`，找出和 ≥ `target` 的长度最小的连续子数组。

```python
def minSubArrayLen(target: int, nums: list[int]) -> int:
    """
    思路：可变窗口
    右指针扩展求和，和≥target时收缩左指针找最小长度
    """
    l = 0
    total = 0
    ans = float('inf')
    
    for r, val in enumerate(nums):
        total += val                     # 扩展窗口
        
        while total >= target:           # 满足条件 → 收缩
            ans = min(ans, r - l + 1)    # 更新最小长度
            total -= nums[l]             # 移出左边界
            l += 1                       # 左指针右移
    
    return 0 if ans == float('inf') else ans
```

**⭐ OD考点**：这是滑动窗口的「Hello World」题，OD基础题中经常出现求和类子数组问题。

**复杂度**：O(n) 时间 | O(1) 空间

**手把手推演**：
```
输入: target=7, nums=[2,3,1,2,4,3]

r=0: total=2   <7 → 继续
r=1: total=5   <7 → 继续
r=2: total=6   <7 → 继续
r=3: total=8  ≥7 → 收缩! ans=min(∞,4)=4
     total-=2=6 <7 → 停止收缩
r=4: total=10 ≥7 → 收缩! ans=min(4,3)=3 ✅
     total-=3=7 ≥7 → 继续! ans=min(3,3)=3
     total-=1=6 <7 → 停止
r=5: total=9  ≥7 → 收缩! ans=min(3,2)=2 ✅🎉
     total-=1=8 ≥7 → ans=min(2,2)=2
     total-=2=6 <7 → 停止
结果: 2
```

### 题2：无重复字符的最长子串（⭐⭐⭐ / OD高频🔥）

**题目**：给定一个字符串 `s`，找出不含重复字符的最长子串的长度。

```python
def lengthOfLongestSubstring(s: str) -> int:
    """
    思路：可变窗口 + 哈希表记录字符最近位置
    遇到重复字符时，左指针跳到重复位置+1
    """
    seen = {}          # 字符 → 最近出现的索引
    l = 0
    ans = 0
    
    for r, ch in enumerate(s):
        # 如果ch出现过且在窗口内 → 跳过重复部分
        if ch in seen and seen[ch] >= l:
            l = seen[ch] + 1
        
        seen[ch] = r                     # 记录/更新位置
        ans = max(ans, r - l + 1)        # 更新最大长度
    
    return ans
```

**⭐ OD考点**：华为OD机考100分题中的**高频题**，几乎每场都会出现类似题！

**复杂度**：O(n) 时间 | O(min(m, n)) 空间（m为字符集大小）

**手把手推演**：
```
输入: s = "abcabcbb"

r=0: ch='a', seen={'a':0},    l=0, 窗口"a"         len=1, ans=1
r=1: ch='b', seen={'a':0,'b':1}, l=0, 窗口"ab"     len=2, ans=2
r=2: ch='c', seen={'a':0,'b':1,'c':2}, l=0, 窗口"abc" len=3, ans=3
r=3: ch='a', seen['a']=0 >= l(0) → l=1
     seen={'a':3,'b':1,'c':2}, 窗口"bca" len=3, ans=3
r=4: ch='b', seen['b']=1 >= l(1) → l=2
     seen={'a':3,'b':4,'c':2}, 窗口"cab" len=3, ans=3
r=5: ch='c', seen['c']=2 >= l(2) → l=3
     seen={'a':3,'b':4,'c':5}, 窗口"abc" len=3, ans=3
r=6: ch='b', seen['b']=4 >= l(3) → l=5
     seen={'a':3,'b':6,'c':5}, 窗口"b" len=1, ans=3
r=7: ch='b', seen['b']=6 >= l(5) → l=7
     seen={'a':3,'b':7,'c':5}, 窗口"b" len=1, ans=3

结果: 3 🎉
```

### 题3：乘积小于K的子数组（⭐⭐）

**题目**：统计乘积严格小于 `k` 的连续子数组个数。

```python
def numSubarrayProductLessThanK(nums: list[int], k: int) -> int:
    """
    思路：可变窗口
    关键技巧：每次加入新元素后，以r结尾的合法子数组有 r-l+1 个
    """
    if k <= 1:         # 乘积为正数，k≤1时不可能有合法子数组
        return 0
    
    l = 0
    prod = 1
    ans = 0
    
    for r, val in enumerate(nums):
        prod *= val                    # 扩展窗口
        
        while prod >= k:               # 乘积≥k时收缩
            prod //= nums[l]
            l += 1
        
        ans += r - l + 1               # 关键：统计以r结尾的所有合法子数组
    
    return ans
```

**复杂度**：O(n) 时间 | O(1) 空间

**💡 技巧解释**：为什么 `ans += r - l + 1`？
```
假设窗口 nums[l..r] 的乘积 < k
那么以 nums[r] 结尾的子数组有：
  nums[l..r], nums[l+1..r], ..., nums[r]  (共 r-l+1 个)
它们的乘积都 < k（因为前缀的乘积更小）！

例：nums=[10,5,2,6], k=100
    窗口 [10,5,2] 乘积=100 >=k → 收缩到 [5,2]
    r=2时，以2结尾的合法子数组：[5,2], [2] → 2个 ✓
```

### 题4：和为K的子数组（⭐⭐⭐ / OD高频🔥）

**题目**：统计和为 `k` 的连续子数组个数。

```python
def subarraySum(nums: list[int], k: int) -> int:
    """
    思路：前缀和 + 哈希表
    """
    from collections import defaultdict
    prefix = defaultdict(int)
    prefix[0] = 1       # 空数组前缀和为0
    cur = 0
    ans = 0
    
    for val in nums:
        cur += val
        ans += prefix[cur - k]    # 找到之前的某个位置，使得差为k
        prefix[cur] += 1
    
    return ans
```

**复杂度**：O(n) 时间 | O(n) 空间

**手把手推演**：
```
输入: nums=[1,2,3], k=3

初始化: prefix={0:1}, cur=0, ans=0

val=1: cur=1, 查 prefix[1-3]=prefix[-2]=0 → ans+=0
       prefix={0:1, 1:1}
val=2: cur=3, 查 prefix[3-3]=prefix[0]=1 → ans+=1
       prefix={0:1, 1:1, 3:1}  → 找到一个: [1,2]
val=3: cur=6, 查 prefix[6-3]=prefix[3]=1 → ans+=1
       prefix={0:1, 1:1, 3:1, 6:1}  → 又找到一个: [3]

结果: ans=2 🎉
```

**⭐ OD考点**：OD 100分题中**必考**！前缀和 + 哈希表是必会套路！

### 题5：最小覆盖子串（⭐⭐⭐⭐ / OD挑战🔥）

**题目**：在字符串 `s` 中找到最短的连续子串，包含字符串 `t` 中的所有字符（含重复）。

```python
def minWindow(s: str, t: str) -> str:
    """
    思路：计数窗口
    need: 每个字符还需要多少个
    missing: 总共还差多少个字符
    """
    from collections import Counter
    need = Counter(t)
    missing = len(t)
    l = 0
    ans_start, ans_len = 0, float('inf')
    
    for r, ch in enumerate(s):
        # 扩展窗口
        if ch in need:
            if need[ch] > 0:      # 有效减少
                missing -= 1
            need[ch] -= 1
        
        # 全部找到 → 收缩找最短
        while missing == 0:
            if r - l + 1 < ans_len:
                ans_start = l
                ans_len = r - l + 1
            
            left_ch = s[l]
            if left_ch in need:
                need[left_ch] += 1
                if need[left_ch] > 0:  # 不够了
                    missing += 1
            l += 1
    
    return "" if ans_len == float('inf') else s[ans_start:ans_start + ans_len]
```

**复杂度**：O(m + n) 时间 | O(k) 空间 (k为t中不同字符数)

**手把手推演**：
```
输入: s="ADOBECODEBANC", t="ABC"

初始: need={'A':1,'B':1,'C':1}, missing=3

r=0: 'A' → need['A']=0>0? yes → missing=2, need={'A':0,'B':1,'C':1}
r=1: 'D' → 不在need中，忽略
r=2: 'O' → 不在need中，忽略
r=3: 'B' → need['B']=0>0? yes → missing=1, need={'A':0,'B':0,'C':1}
r=4: 'E' → 忽略
r=5: 'C' → need['C']=0>0? yes → missing=0! ✅ "ADOBEC" 覆盖了
       收缩: len=6, ans="ADOBEC"
       移出'A': need['A']=1>0? yes → missing=1 → 停止收缩
r=6: 'O' → 忽略 (继续，missing=1)
r=7: 'D' → 忽略
r=8: 'E' → 忽略
r=9: 'B' → need['B']=-1→-2 (多余的B), missing仍=1
r=10: 'A' → need['A']=0>0? yes → missing=0! ✅ "CODEBA"
        收缩: len=5<6, ans="CODEBA"
        移出'C': need['C']=1>0? yes → missing=1 → 停止
r=11: 'N' → 忽略
r=12: 'C' → need['C']=0>0? yes → missing=0! ✅ "BANC"
        收缩: len=4<5, ans="BANC" 🎉
        移出'B': need['B']=0>0? yes → missing=1 → 停止

结果: "BANC" ✅
```

**⭐ OD考点**：OD 200分题难度，但思路清晰后其实不难！关键是理解 `missing` 和 `need` 的配合。

### 题6：最大连续1的个数III（⭐⭐）

**题目**：最多可以把 `k` 个0翻转为1，求最长连续1的长度。

```python
def longestOnes(nums: list[int], k: int) -> int:
    """
    思路：窗口内最多允许k个0
    0的数量超过k时收缩左指针
    """
    l = 0
    zeros = 0
    ans = 0
    
    for r, val in enumerate(nums):
        if val == 0:
            zeros += 1           # 记录窗口内0的个数
        
        while zeros > k:         # 0太多 → 收缩
            if nums[l] == 0:
                zeros -= 1
            l += 1
        
        ans = max(ans, r - l + 1)
    
    return ans
```

**复杂度**：O(n) 时间 | O(1) 空间

**💡 换个思路**：你也可以把问题理解成「找最长的子数组，其中0的个数≤k」。

### 题7：连续数组（⭐⭐⭐）

**题目**：二进制数组中，含有相同数量0和1的最长连续子数组长度。

```python
def findMaxLength(nums: list[int]) -> int:
    """
    思路：0→-1，1→+1，找和为0的最长子数组
    前缀和+哈希表存首次出现的位置
    """
    first = {0: -1}    # 前缀和为0首次出现在索引-1（空数组）
    cur = 0
    ans = 0
    
    for i, val in enumerate(nums):
        cur += 1 if val == 1 else -1    # 把0当成-1
        
        if cur in first:
            ans = max(ans, i - first[cur])  # 相同前缀和 → 中间段和为0
        else:
            first[cur] = i              # 首次出现，记录位置
    
    return ans
```

**复杂度**：O(n) 时间 | O(n) 空间

**可视化**：
```
输入: nums=[0,1,0]

映射: 0→-1, 1→+1 → [-1, +1, -1]
前缀和: 0, -1, 0, -1

索引:   -1   0    1    2
前缀和:  0  -1    0   -1
          ↑         ↑
          前缀和相同！中间段和=0
          [0,1] = 1个0+1个1 → len=2 ✅
```

## 五、性能优化与常见坑

### 5.1 Python性能陷阱

#### ❌ 陷阱1：在滑动窗口内使用 O(n) 的查找
```python
# ❌ 每次检查窗口内是否有重复 → O(n) 每次 = O(n²)
def bad(s):
    for r in range(len(s)):
        while len(set(s[l:r+1])) < r-l+1:  # set()创建切片O(k)
            l += 1

# ✅ 用哈希表记录位置的查找是 O(1)
def good(s):
    seen = {}
    for r, ch in enumerate(s):
        if ch in seen and seen[ch] >= l:  # O(1)
            l = seen[ch] + 1
```

#### ❌ 陷阱2：使用 list 作为计数器
```python
# ❌ 计数时每次都 list.count() → O(k)
for ch in window:
    if window.count(ch) ...

# ✅ 用 Counter/dict → O(1)
need = Counter(t)
need[ch] -= 1  # O(1)
```

#### ❌ 陷阱3：前缀和用列表每次求和
```python
# ❌ 暴力 O(n²)
for i in range(n):
    for j in range(i, n):
        if sum(nums[i:j+1]) == k:  # sum() O(n)

# ✅ 前缀和 O(n)
prefix = defaultdict(int)
for val in nums:
    cur += val
    ans += prefix[cur - k]
```

### 5.2 常见边界条件

| 问题 | 边界条件 | 处理方式 |
|------|---------|---------|
| 空数组/空字符串 | s=="" | 直接返回0或"" |
| k≤1的乘积问题 | k≤1 | 直接返回0（乘积≥1） |
| 找不到符合条件的子串 | 无解 | 返回0或""（取决于题目） |
| 滑动窗口左指针越界 | l>r | 检查seen[ch]≥l的条件 |
| 负数存在时不能用滑动窗口 | 数组有负数 | 改用前缀和+哈希表 |

### 5.3 小白问：什么时候不能用滑动窗口？

**答：当数组中存在负数时！** 因为负数会让窗口的和/乘积无法单调判断——加入负数反而会减少和，让收缩条件失效。这时只能使用前缀和+哈希表方法。
