## 二、核心知识点详解

### 2.1 什么是异位词（Anagram）？

**异位词** = 两个字符串包含的字符**种类和数量相同**，但排列顺序不同。

```
"abc" 和 "cba" → 是异位词 ✅
"abc" 和 "abd" → 不是异位词 ❌（字符种类不同）
"abc" 和 "aabbc" → 不是异位词 ❌（字符数量不同）
```

**生活类比**：你和你朋友都买了晨光文具的12色彩色笔，虽然你们笔盒里笔的摆放顺序不同，但笔的种类和数量完全一样 👉 这就是异位词！

**核心判断方法**：比较两个字符串的**字符频次数组**是否相等。

### 2.2 什么是原地压缩？

**原地压缩** = 直接在原数组上修改，不额外开新数组空间。

生活类比：你在整理书架，要把同一本书的复制本合并成一摞，上面贴个标签写"×3"，你直接在这个书架上操作，不把书搬到另一个房间去整理。

**关键在于**：用一个"读指针"从头到尾扫描，一个"写指针"指示当前写到哪了。

### 2.3 什么是贪心算法？

**贪心算法** = 每一步都选当前看起来最好的，不回头看。

生活类比：你逛自助餐，每次只选桌上最贵的菜吃，不管后面还有没有更好的 👉 这就是贪心。有时候这能让你吃到最贵的，有时候你会后悔前面吃太饱错过了后面的龙虾 🦞

**在递增三元组问题中**：我们维护两个值——当前最小值（first）和当前第二小的值（second），每走一步都更新它们。这就是贪心！

### 2.4 什么是拓扑排序？

**拓扑排序** = 给有向无环图的节点排个序，使得每条边的起点都在终点前面。

生活类比：大学选课——你要先学"高等数学"才能学"线性代数"，先学"线性代数"才能学"机器学习"...

```
高数 → 线代 → 机器学习
```

拓扑排序的结果就是：高数 → 线代 → 机器学习

**在字符串问题中的应用**：外星人说的单词里隐藏着他们的字母顺序规则！两个单词 `"wrt", "wrf"` 比较，第一个不同的字符是 `'t'` 和 `'f'`，说明 `'t'` 在 `'f'` 前面。

### 2.5 什么叫环状最小差值？

**环状** = 首尾相连，就像一个圆环。

时间也是环状的：23:59 和 00:00 只差1分钟（因为过了23:59就是00:00）。

处理环状问题的通用方法：
1. 把所有值转换成同一单位（比如分钟）
2. 排序
3. 计算相邻差值
4. **别忘了**处理首尾的环状差值！`1440 - 最后一个 + 第一个`

## 四、核心套路详解

### 套路1：滑动窗口 + 计数数组 — 异位词匹配

#### 🎯 识别特征

看到题目中出现以下关键词，立刻想到这个套路：
- "异位词"、"字母异位词"、"anagram"
- "s 中所有 p 的子串"
- 固定窗口大小（len(p)）
- 只包含小写字母（可以用26大小的数组）

#### 📝 模板代码

```python
def findAnagrams(s: str, p: str) -> list[int]:
    """
    在 s 中找到所有 p 的异位词子串的起始索引
    模板：固定滑动窗口 + 计数数组
    """
    n, m = len(s), len(p)
    if n < m:
        return []  # 窗口比整个字符串还大，不可能找到
    
    # 初始化：只有小写字母，所以用长度为26的数组
    cnt_p = [0] * 26  # p 的字符频次
    cnt_w = [0] * 26  # 当前窗口的字符频次
    a = ord('a')       # 基准ASCII
    
    for ch in p:
        cnt_p[ord(ch) - a] += 1  # 统计 p 中每个字符
    
    res = []
    for i, ch in enumerate(s):
        # 窗口右边界右移 → 加入新字符
        cnt_w[ord(ch) - a] += 1
        
        # 窗口左边界右移 → 移除旧字符
        if i >= m:
            cnt_w[ord(s[i - m]) - a] -= 1
        
        # 判断窗口是否匹配
        if cnt_w == cnt_p:
            res.append(i - m + 1)  # 窗口起始索引
    
    return res
```

#### ✋ 手把手推演

以 `s = "cbaebabacd", p = "abc"` 为例：

```
p = "abc" → cnt_p = [1,1,1,0,0,...]  (a=1, b=1, c=1)
m = 3, 窗口大小 = 3

第1步: i=0, ch='c'
  cnt_w = [0,0,1,0,0,...]
  i < 3 不移除
  cnt_w != cnt_p → 不匹配

第2步: i=1, ch='b'
  cnt_w = [0,1,1,0,0,...]
  不匹配

第3步: i=2, ch='a'
  cnt_w = [1,1,1,0,0,...]
  cnt_w == cnt_p → 匹配! 起始索引 i-m+1 = 0 ✅
  res = [0]

第4步: i=3, ch='e'
  加入 'e': cnt_w = [1,1,1,1,0,...]
  移除 s[0]='c': cnt_w = [1,1,0,1,0,...]
  不匹配

...以此类推，i=6时匹配 → res = [0, 6]
```

#### 📊 适用范围

| 场景 | 适用性 | 说明 |
|------|--------|------|
| 小写字母 | ✅ 完美 | 26数组，O(1)比较 |
| ASCII字符 | ✅ 适用 | 128/256数组 |
| Unicode字符 | ❌ 不适用 | 用Counter或字典 |
| 需要高级数据结构 | ❌ 不用 | 普通题用数组就够 |

### 套路2：双指针原地读写 — 字符串压缩

#### 🎯 识别特征

- "原地"、"in-place"、"常量额外空间"
- "压缩"、"压缩字符串"
- "连续重复字符"、"连续相同"
- 需要在原数组上修改并返回新长度

#### 📝 模板代码

```python
def compress(chars: list[str]) -> int:
    """
    原地压缩字符数组
    模板：读指针扫描 + 写指针写入
    返回：压缩后的新长度
    """
    write = 0    # 写指针：下一个写入位置
    i = 0        # 读指针：当前位置
    n = len(chars)
    
    while i < n:
        ch = chars[i]
        cnt = 1
        
        # 统计连续相同字符的个数
        while i + cnt < n and chars[i + cnt] == ch:
            cnt += 1
        
        # 写入字符
        chars[write] = ch
        write += 1
        
        # 如果个数 > 1，写入数字
        if cnt > 1:
            for c in str(cnt):   # 注意！cnt可能是多位数（如12）
                chars[write] = c
                write += 1
        
        # 读指针跳到下一组
        i += cnt
    
    return write
```

#### ✋ 手把手推演

以 `chars = ["a","a","b","b","c","c","c"]` 为例：

```
初始状态:
  chars = [a, a, b, b, c, c, c, ...]
           ↑
           i=0, write=0

第1轮: i=0, ch='a', cnt=2
  chars[0] = 'a', write=1
  cnt>1 → chars[1] = '2', write=2
  i += 2 → i=2

  chars = [a, 2, b, b, c, c, c, ...]
                  ↑
                  i=2, write=2

第2轮: i=2, ch='b', cnt=2
  chars[2] = 'b', write=3
  chars[3] = '2', write=4
  i += 2 → i=4

  chars = [a, 2, b, 2, c, c, c, ...]
                        ↑
                        i=4, write=4

第3轮: i=4, ch='c', cnt=3
  chars[4] = 'c', write=5
  chars[5] = '3', write=6
  i += 3 → i=7

  chars = [a, 2, b, 2, c, 3, c, ...]
                                    ↑
                                    i=7(结束)

返回 write=6 ✅
```

> 💡 **多位数处理**：当 cnt=12 时，`str(12)` = `"12"`，会逐个写入 '1' 和 '2'。这就是为什么要用 `for c in str(cnt)` 循环！

### 套路3：贪心维护极值 — 递增三元组

#### 🎯 识别特征

- "递增子序列"、"递增三元组"
- "是否存在长度为3的递增..."
- "O(n)时间复杂度，O(1)空间复杂度"

#### 📝 模板代码

```python
def increasingTriplet(nums: list[int]) -> bool:
    """
    判断是否存在长度为3的递增子序列
    模板：贪心维护 first(最小值) 和 second(第二小值)
    """
    first = second = float('inf')  # 初始化为正无穷
    
    for x in nums:
        if x <= first:
            # 遇到更小的值，更新 first
            first = x
        elif x <= second:
            # 介于 first 和 second 之间，更新 second
            second = x
        else:
            # 比 first 和 second 都大，找到了！
            return True
    
    return False
```

#### ✋ 手把手推演

以 `nums = [5, 1, 5, 5, 2, 5, 4]` 为例：

```
初始: first=∞, second=∞

第1步: x=5
  x <= first(∞) → first=5
  状态: first=5, second=∞

第2步: x=1
  x <= first(5) → first=1
  状态: first=1, second=∞

第3步: x=5
  x > first(1), x <= second(∞) → second=5
  状态: first=1, second=5

第4步: x=5
  x > first(1), x <= second(5) → second=5 (不变)
  状态: first=1, second=5

第5步: x=2
  x > first(1), x <= second(5) → second=2
  状态: first=1, second=2

第6步: x=5
  x > first(1), x > second(2) → 找到! return True ✅
  (1, 2, 5)
```

> 💡 **这个算法的巧妙之处**：虽然 first 和 second 不一定是同一个递增子序列的前两个元素（比如这里 first=1, second=2 其实是来自不同位置的），但它们的意义是：
> - **first** = 到目前为止的最小值
> - **second** = 到目前为止**在某个 first 之后**的最小第二小值
>
> 只要找到一个比 second 大的数，就一定存在递增三元组！🎉

#### 🧪 证明

假设我们找到了 `x > second`：
1. second 是在某个 first' 之后更新的（在 first' 之后发现了比它大但比当前 second 小的数）
2. first' ≤ first... 不，更准确的说法是：`first` 一定在 `second` 之前出现，且在 `second` 之前
3. 所以顺序是：`first → second → x`，且 `first < second < x` ✅

### 套路4：图论建图 + 拓扑排序 — 外星词典

#### 🎯 识别特征

- "外星词典"、"外星文字"、"Alien Dictionary"
- "新字母顺序"、"从单词列表中推导顺序"
- 相邻单词比较 + 有向无环图(DAG)

#### 📝 模板代码

```python
from collections import defaultdict, deque

def alienOrder(words: list[str]) -> str:
    """
    从外星单词列表中推导字母顺序
    模板：建图(邻接表) + 拓扑排序(BFS Kahn算法)
    """
    # 第一步：初始化所有出现的字母
    g = defaultdict(set)    # 邻接表
    indeg = defaultdict(int)  # 入度表
    
    for w in words:
        for ch in w:
            indeg.setdefault(ch, 0)  # 每个字母先初始化入度为0
    
    # 第二步：比较相邻单词，构建有向边
    for i in range(len(words) - 1):
        w1, w2 = words[i], words[i + 1]
        
        # 非法情况：w1比w2长，且w1以w2为前缀
        # 如 "abc" 在 "ab" 前面 → 不可能！
        if len(w1) > len(w2) and w1[:len(w2)] == w2:
            return ""
        
        # 找第一个不同字符
        for c1, c2 in zip(w1, w2):
            if c1 != c2:
                if c2 not in g[c1]:     # 避免重复边
                    g[c1].add(c2)       # c1 → c2
                    indeg[c2] += 1      # c2 入度+1
                break  # 只比较第一个不同字符就够
    
    # 第三步：拓扑排序（BFS）
    q = deque([c for c, d in indeg.items() if d == 0])
    res = ""
    
    while q:
        c = q.popleft()
        res += c
        for nxt in g[c]:
            indeg[nxt] -= 1
            if indeg[nxt] == 0:
                q.append(nxt)
    
    # 如果结果字符串包含所有字母，说明成功
    return res if len(res) == len(indeg) else ""
```

#### ✋ 手把手推演

以 `words = ["wrt", "wrf", "er", "ett", "rftt"]` 为例：

```
第一步：所有出现过的字符
  indeg = {w:0, r:0, t:0, f:0, e:0}

第二步：比较相邻单词，找规则
  比较 "wrt" 和 "wrf":
    w=w, r=r, t≠f → t在f前面 → t→f
    indeg = {w:0, r:0, t:0, f:1, e:0}

  比较 "wrf" 和 "er":
    w≠e → w在e前面 → w→e
    indeg = {w:0, r:0, t:0, f:1, e:1}

  比较 "er" 和 "ett":
    e=e, r≠t → r在t前面 → r→t
    indeg = {w:0, r:0, t:1, f:1, e:1}

  比较 "ett" 和 "rftt":
    e≠r → e在r前面 → e→r
    indeg = {w:0, r:1, t:1, f:1, e:1}

  图结构:
    t → f
    w → e
    r → t
    e → r

  入度表:
    w:0, e:1, r:1, t:1, f:1

第三步：拓扑排序
  队首入度为0的: [w]
  出队 w → res="w", e入度-1=0 → 入队 [e]
  出队 e → res="we", r入度-1=0 → 入队 [r]
  出队 r → res="wer", t入度-1=0 → 入队 [t]
  出队 t → res="wert", f入度-1=0 → 入队 [f]
  出队 f → res="wertf"

结果: "wertf" ✅
```

> **小白问**：为什么要比较相邻单词？不能比较任意两个单词吗？
>
> **答**：相邻单词之间的差异能给出确定性的字母顺序。如果比较第1个和第3个单词，它们之间可能隔着多个字母的变化，推导出的关系不一定准确。相邻比较是最保险的——只取第一个不同字符，这个顺序一定是确定的！

### 题1：找到字符串中所有字母异位词

#### 📝 题目描述

给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的**异位词**子串，返回这些子串的起始索引。

**示例 1：**
```
输入: s = "cbaebabacd", p = "abc"
输出: [0, 6]
解释:
起始索引 0 → "cba" → "abc" 的异位词 ✅
起始索引 6 → "bac" → "abc" 的异位词 ✅
```

**示例 2：**
```
输入: s = "abab", p = "ab"
输出: [0, 1, 2]
解释:
起始索引 0 → "ab" ✅
起始索引 1 → "ba" ✅
起始索引 2 → "ab" ✅
```

#### 💡 解题思路

这题的**核心**就是我们在套路1中讲的——固定窗口滑动 + 计数数组比较。

但这里有个可以优化的地方：我们不需要每次都完整比较两个数组（O(26)），可以用一个计数器 `diff` 来跟踪差异个数：

```python
def findAnagrams_optimized(s: str, p: str) -> list[int]:
    n, m = len(s), len(p)
    if n < m:
        return []
    
    cnt = [0] * 26
    a = ord('a')
    
    # 先把 p 全部加到 cnt，把 s 的前 m 个从 cnt 减去
    for i in range(m):
        cnt[ord(p[i]) - a] += 1
        cnt[ord(s[i]) - a] -= 1
    
    # 统计初始差异
    diff = sum(1 for x in cnt if x != 0)
    
    res = []
    if diff == 0:
        res.append(0)
    
    # 滑动窗口
    for i in range(m, n):
        # 移除左边界字符
        idx = ord(s[i - m]) - a
        cnt[idx] += 1
        if cnt[idx] == 1:    # 从0变1 → 多了一个差异
            diff += 1
        elif cnt[idx] == 0:  # 从-1变0 → 少了一个差异
            diff -= 1
        
        # 加入右边界字符
        idx = ord(s[i]) - a
        cnt[idx] -= 1
        if cnt[idx] == -1:   # 从0变-1 → 多了一个差异
            diff += 1
        elif cnt[idx] == 0:  # 从1变0 → 少了一个差异
            diff -= 1
        
        if diff == 0:
            res.append(i - m + 1)
    
    return res
```

#### ⏱ 复杂度分析

| 实现方式 | 时间复杂度 | 空间复杂度 |
|----------|-----------|-----------|
| 基础版（完整比较数组） | O(n × 26) = O(n) | O(1) |
| 优化版（diff追踪） | O(n) | O(1) |

给你一个字符数组 `chars`，请原地压缩：
- 连续相同的字符 → 字符 + 次数（次数为1时不写数字）
- 返回压缩后的新长度

**示例：**
```
输入: chars = ["a","a","b","b","c","c","c"]
输出: 6

1. **多位数处理**：cnt=12时，要逐个写入 '1' 和 '2'

  cnt>1 → str(12)="12"
    chars[2]='1', write=3
    chars[3]='2', write=4
  i += 12 → i=13(结束)

返回 write=4
```

### 题3：递增的三元子序列

#### 📝 题目描述

给你一个整数数组 `nums`，判断是否存在长度为3的递增子序列 `(i < j < k)` 使得 `nums[i] < nums[j] < nums[k]`。

**示例：**
```
输入: nums = [1,2,3,4,5]
输出: true (1,2,3)

输入: nums = [5,4,3,2,1]
输出: false

输入: nums = [2,1,5,0,4,6]
输出: true (0,4,6 或 0,2,6 等)
```

#### 💡 解题思路

就是套路3！贪心维护 first（最小值）和 second（second小值）。

> **小白问**：有没有可能 first 和 second 不是同一个递增序列的？比如 first 在非常靠前的位置，second 在靠后的位置，它们之间夹着一个比 second 大的数吗？
>
> **答**：这正是这个算法的精髓！就算 first 和 second 来自"不同路径"，只要存在任何一个数 x > second，就说明在 second 之前一定有一个比 second 小的数（就是那个更新 second 时的 first'），所以一定存在 first' < second < x。这就是**数学归纳法**的思想！

|------|-----|
| 时间复杂度 | O(n) — 一次遍历 |
| 空间复杂度 | O(1) — 两个变量 |

### 题4：最小时间差

#### 📝 题目描述

给定一个24小时制的时间列表（"HH:MM"格式），找出任意两个时间的最小时间差（分钟数）。

**示例：**
```
输入: timePoints = ["23:59","00:00"]
输出: 1 (23:59到00:00只差1分钟)

输入: timePoints = ["00:00","23:59","00:00"]
输出: 0 (有重复时间)
```

#### 💡 解题思路

突破点：**把所有时间统一到分钟**！

```
"23:59" → 23×60 + 59 = 1439 分钟
"00:00" → 0×60 + 0 = 0 分钟
```

然后排序，相邻比较，最后别忘了**环状比较**（首尾差值）。

```python
def findMinDifference(timePoints: list[str]) -> int:
    # 转换为分钟
    minutes = []
    for t in timePoints:
        h, m = map(int, t.split(":"))
        minutes.append(h * 60 + m)
    
    # 排序
    minutes.sort()
    n = len(minutes)
    
    ans = float('inf')
    
    # 比较相邻
    for i in range(n - 1):
        ans = min(ans, minutes[i + 1] - minutes[i])
        if ans == 0:  # 提前退出
            return 0
    
    # 环状：首尾差值
    ans = min(ans, 1440 - minutes[-1] + minutes[0])
    
    return ans
```

#### ✋ 推演

```
timePoints = ["23:59", "00:00"]
→ 分钟 = [1439, 0]
→ 排序 = [0, 1439]
  相邻差: 1439 - 0 = 1439
  环状: 1440 - 1439 + 0 = 1 ✅  min(1439, 1) = 1
```

#### ⏱ 复杂度分析

| 维度 | 值 |
|------|-----|
| 时间复杂度 | O(n log n) — 排序是瓶颈 |
| 空间复杂度 | O(n) — 存储分钟数组 |

#### 💡 进阶优化

如果时间数量 > 1440，根据**鸽巢原理**，一定有重复时间，答案就是0！

```python
def findMinDifference_optimized(timePoints: list[str]) -> int:
    if len(timePoints) > 1440:  # 超过1440个时间，必有重复
        return 0
    
    # ... 同上
```

### 题5：文本左右对齐

#### 📝 题目描述

给定一个单词数组 `words` 和一个长度 `maxWidth`，重新排版成每行恰好 `maxWidth` 个字符的左右对齐文本。

规则：
1. 每行尽量多地放单词（贪心）
2. 单词间空格要**均匀分配**，左侧空格数 ≥ 右侧
3. 最后一行**左对齐**（单词间距为1，右侧补满空格）
4. 只有一个单词的行也左对齐

**示例：**
```
输入: words = ["This", "is", "an", "example", "of", "text", "justification."], maxWidth = 16
输出:
[
  "This    is    an",   ← 空格均匀分配(3+4)
  "example  of text",   ← 空格均匀分配(2+3)
  "justification.  "    ← 左对齐，右侧补到16
]
```

#### 💡 解题思路

这题思路很直接，但**细节很多**，是典型的模拟题：

1. **分组**：贪心地往每行塞单词，直到塞不下下一个
2. **对齐**：
   - 最后一行或只有一个单词 → 左对齐（单词间1个空格，右侧补满）
   - 其他行 → 均匀分配空格

**均匀分配空格的算法**：
- `spaces = maxWidth - 所有单词长度之和` ← 总共需要这么多空格
- `each = spaces // (单词数 - 1)` ← 每个间隔的基本空格数
- `extra = spaces % (单词数 - 1)` ← 前 extra 个间隔各多一个空格

#### 🎯 模板代码

```python
def fullJustify(words: list[str], maxWidth: int) -> list[str]:
    res = []
    i = 0
    n = len(words)
    
    while i < n:
        # 步骤1：确定当前行放哪些单词
        j = i
        cur_len = 0
        while j < n and cur_len + len(words[j]) + (j - i) <= maxWidth:
            # (j - i) 是最少需要的间隔空格数
            cur_len += len(words[j])
            j += 1
        
        words_cnt = j - i          # 当前行的单词数
        spaces = maxWidth - cur_len  # 需要填充的空格总数
        
        # 步骤2：对齐
        # 情况1：最后一行 或 只有一个单词 → 左对齐
        if j == n or words_cnt == 1:
            line = " ".join(words[i:j])
            line += " " * (maxWidth - len(line))
        else:
            # 情况2：均匀分配
            each = spaces // (words_cnt - 1)       # 基本空格数
            extra = spaces % (words_cnt - 1)       # 多出的空格数
            line = ""
            for k in range(words_cnt - 1):
                line += words[i + k]
                line += " " * (each + (1 if k < extra else 0))
            line += words[j - 1]  # 最后一个单词后面不加空格
        
        res.append(line)
        i = j  # 处理下一行
    
    return res
```

#### ✋ 推演

以 `words = ["This", "is", "an", "example", "of", "text", "justification."], maxWidth = 16`：

```
第1行:
  i=0, 尝试放 "This" → OK (4+0=4)
  尝试放 "This is" → OK (4+2+1=7)
  尝试放 "This is an" → OK (4+2+2+2=10)
  尝试放 "This is an example" → ❌ (4+2+2+7+3=18 > 16)
  words = ["This","is","an"], word_cnt=3, chars=4+2+2=8
  spaces = 16-8 = 8
  each = 8//2 = 4, extra = 8%2 = 0
  line = "This" + "    " + "is" + "    " + "an"
  → "This    is    an" ✅

第2行:
  i=3, 尝试放 "example" → OK (7+0=7)
  尝试放 "example of" → OK (7+2+1=10)
  尝试放 "example of text" → OK (7+2+4+2=15)
  尝试放 "example of text justification." → ❌ 
  words = ["example","of","text"], word_cnt=3, chars=7+2+4=13
  spaces = 16-13 = 3
  each = 3//2 = 1, extra = 3%2 = 1
  line = "example" + "  " + "of" + " " + "text"
  → "example  of text" ✅

第3行 (最后一行):
  i=6, word = ["justification."] (只有一个)
  words_cnt=1, 左对齐
  line = "justification." + "  " (2个空格补到16)
  → "justification.  " ✅
```

#### ⏱ 复杂度分析

| 维度 | 值 |
|------|-----|
| 时间复杂度 | O(n) — 每个单词处理一次 |
| 空间复杂度 | O(1) — 不计算输出空间 |

### 题6：火星词典 / 外星词典

#### 📝 题目描述

有一个外星语言，字母顺序与英语不同。给定一个按**外星字典序**排序的单词列表，推导出外星字母的顺序。

**示例：**
```
输入: words = ["wrt","wrf","er","ett","rftt"]
输出: "wertf"

输入: words = ["z","x"]
输出: "zx"

输入: words = ["z","x","z"]
输出: "" ← 矛盾，不可能
```

#### 💡 解题思路

这就是套路4——**拓扑排序**！三步走：

1. **建图**：比较相邻单词，找到第一个不同的字符对 (c1, c2)，建立有向边 c1 → c2
2. **拓扑排序**：BFS（Kahn算法）输出入度为0的节点
3. **检测环**：如果结果字符串长度 ≠ 字母总数，说明有环 → 矛盾

#### 🎯 特殊情况处理

- **前缀冲突**：`["abc", "ab"]` → "abc" 在 "ab" 前面，但 "abc" 以 "ab" 开头且更长，这不可能！
- **重复边**：多对单词可能推导出相同的边，要用 set 去重
- **孤立节点**：某个字母可能只出现一次，没有与其他字母比较，直接加到入度为0的队列

#### ⏱ 复杂度分析

| 维度 | 值 |
|------|-----|
| 时间复杂度 | O(C) — C是所有单词的总字符数 |
| 空间复杂度 | O(1) — 最多26个字母 |

## 六、性能优化与常见坑

### ❌ 坑1：字符串拼接的性能陷阱

```python
# ❌ 错误：循环拼接，O(n²)！
result = ""
for word in words:
    result += word + " "  # 每次拼接都创建新字符串

# ✅ 正确：用 join，O(n)
result = " ".join(words)
```

### ❌ 坑2：计数器用字典还是数组？

```python
# 场景：只包含小写字母
# ❌ 用 Counter (开销大)
from collections import Counter
if Counter(s[i:i+m]) == Counter(p):  # 每次 O(m) 创建字典

# ✅ 用长度为26的数组（O(1)比较，O(1)空间）
cnt_p = [0] * 26
```

**选择原则**：
- 字符集有限（如小写字母）→ 用**数组**
- 字符集未知或很大（如Unicode）→ 用 **Counter** 或 **defaultdict**

### ❌ 坑3：滑动窗口的边界条件

```python
# ❌ 忘记处理窗口起步阶段（i < m 时不移除左边）
cnt_w[ord(s[i - m]) - a] -= 1  # 当 i < m 时索引负！

# ✅ 加条件判断
if i >= m:
    cnt_w[ord(s[i - m]) - a] -= 1
```

### ❌ 坑4：压缩时的多位数处理

```python
# ❌ 错误：假设次数总是个位数
chars[write] = str(cnt)  # cnt=12 → 写入了'12'两个字符到一个位置！

# ✅ 正确：逐位写入
for c in str(cnt):
    chars[write] = c
    write += 1
```

### ❌ 坑5：时间差忘记环状处理

```python
# ❌ 错误：只比较相邻，忘掉首尾
ans = min(ans, minutes[i+1] - minutes[i])  # 23:59和00:00的差被忽略！

# ✅ 正确：加上环状比较
ans = min(ans, 1440 - minutes[-1] + minutes[0])
```

### ❌ 坑6：外星词典的前缀冲突

```python
# ❌ 错误：没检查"ab"在"abc"前面的非法情况
# "abc" 在 "ab" 前面 → 不可能！因为在字典序中短单词应该在前面

# ✅ 正确：检查
if len(w1) > len(w2) and w1[:len(w2)] == w2:
    return ""  # 非法，矛盾
```

### ✅ 性能优化一览

| 场景 | ❌ 坏写法 | ✅ 好写法 | 提升 |
|------|----------|----------|------|
| 拼接字符串 | `+` 循环 | `"".join()` | O(n²) → O(n) |
| 字符计数 | `s.count(c)` | `[0]*26`数组 | O(n²) → O(n) |
| 数组比较 | 循环对比 | 直接 `==` | O(n) → O(1) |
| 提前退出 | 全量计算 | 鸽子洞原理 | 最好O(1) |
| 重复边 | list/追加 | set去重 | 避免重复统计 |

5. **测试样例**：写完代码先跑自己构造的边界测试
