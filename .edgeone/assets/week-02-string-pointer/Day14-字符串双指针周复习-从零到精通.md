### 2.1 字符串基础操作大全

> 💡 **小白问："字符串操作到底有哪些？"**
> **答**：Python的字符串就像一把瑞士军刀，基础操作只有十几种，但组合起来能解决90%的问题！

#### 📋 常用操作速查表

| 操作 | 写法 | 示例 | 结果 | 复杂度 |
|------|------|------|------|--------|
| 索引访问 | `s[i]` | `s[0]` | 第1个字符 | O(1) |
| 切片 | `s[l:r]` | `s[1:4]` | 索引1~3的子串 | O(k) |
| 反转 | `s[::-1]` | `"abc"[::-1]` | `"cba"` | O(n) |
| 拼接 | `s1 + s2` 或 `"".join(list)` | `"a"+"b"` | `"ab"` | ⚠️+是O(n²)用join |
| 查找 | `s.find(t)` | `"hello".find("ll")` | `2` | O(n) |
| 计数 | `s.count(ch)` | `"hello".count("l")` | `2` | O(n) |
| 替换 | `s.replace(a, b)` | `"aba".replace("a","x")` | `"xbx"` | O(n) |
| 分割 | `s.split(sep)` | `"a,b".split(",")` | `["a","b"]` | O(n) |
| 大小写 | `s.lower()` / `s.upper()` | `"Ab".lower()` | `"ab"` | O(n) |
| 判断数字 | `ch.isdigit()` | `"5".isdigit()` | `True` | O(1) |
| 判断字母 | `ch.isalpha()` | `"a".isalpha()` | `True` | O(1) |
| 判断空格 | `ch.isspace()` | `" ".isspace()` | `True` | O(1) |
| 判断字母数字 | `ch.isalnum()` | `"a1".isalnum()` | `True` | O(1) |
| 转列表 | `list(s)` | `list("ab")` | `["a","b"]` | O(n) |
| 列表转字符串 | `"".join(list)` | `"".join(["a","b"])` | `"ab"` | O(n) |

#### ⚠️ 三大常见坑

**坑1：循环拼接字符串 → O(n²)**

```python
# ❌ 坏写法 — 每次拼接都创建新字符串
result = ""
for ch in s:
    result += ch          # O(n²)！

# ✅ 好写法 — 收集到列表再join
chars = []
for ch in s:
    chars.append(ch)
result = "".join(chars)   # O(n)
```

**坑2：字符串不可变 — 修改字符用列表**

```python
# ❌ 报错！
s = "hello"
s[0] = "H"   # TypeError: 'str' object does not support item assignment

# ✅ 转为列表修改
chars = list(s)
chars[0] = "H"
s = "".join(chars)   # "Hello"
```

**坑3：空字符串和空格的判断**

```python
s = "   "
bool(s)         # True（有空格）
s.strip() == "" # True（正确判断）
len(s) == 0     # False（正确，但空格不算空）
```

### 2.2 双指针技法

双指针本质是用 **两个游标同时扫描**，把暴力二重循环 `O(n²)` 降到 `O(n)`。

```
暴力：for i in range(n):
           for j in range(n):   ← O(n²)
               ...

双指针：i = 0, j = n-1
         while i < j:          ← O(n)
             ...
```

#### 🎭 两种形态

| 形态 | 指针位置 | 移动规则 | 代表题 |
|------|---------|---------|--------|
| **对撞指针** | 左→右← | 根据条件移动左或右 | 两数之和、三数之和、盛水 |
| **快慢指针** | 同向移动 | 快指针探路，慢指针追 | 移动零、去重、环形链表 |

**对撞指针图解**：

```
初始：[⬅L=0]  .........  [R=n-1=➡]
        ↓                    ↓
循环：while L < R:
        根据条件移动 L++ 或 R--
```

**快慢指针图解**：

```
初始：[⬅S=0][➡F=0] .........
        ↓     ↓
循环：while F < n:
        根据条件选择性移动 S
        F++  # F 一定每步都前进
```

### 2.3 滑动窗口算法

滑动窗口是双指针的子类——**窗口的左右边界都在动**，常用于子串/子数组问题。

#### 📐 三种窗口模板

| 类型 | 特征 | 窗口变化 | 代表题 |
|------|------|---------|--------|
| **固定窗口** | 窗口大小固定为k | 右进一个，左出一个 | 大小为k的平均值 |
| **可变窗口** | 窗口大小可变，满足条件时扩张 | 右指针扩张，左指针收缩 | 无重复字符最长子串 |
| **计数窗口** | 窗口需要满足字符计数条件 | 用Counter/hash跟踪 | 最小覆盖子串、异位词 |

#### 🖼️ 可变窗口动画

```
s = "abcabcbb"

窗口: [a]         seen={a:0}, ans=1
窗口: [ab]        seen={a:0,b:1}, ans=2
窗口: [abc]       seen={a:0,b:1,c:2}, ans=3
窗口: [bca]       a重复！L跳到seen[a]+1=1, ans=3
窗口: [bcab]?     b重复！L跳到seen[b]+1=2, 窗口=[cab], ans=3
窗口: [cabc]?     c重复！L跳到seen[c]+1=3, 窗口=[abc], ans=3
窗口: [abcb]?     b重复！L跳到seen[b]+1=4, 窗口=[cb], ans=3
窗口: [cbb]?      b重复！L跳到seen[b]+1=5, 窗口=[b], ans=3
最终：ans=3（子串"abc"）
```

### 2.4 回文问题

回文（Palindrome）是正反读都一样的字符串，如 `"abccba"`、`"racecar"`。

#### 两种解题路线

| 路线 | 方法 | 复杂度 | 适用场景 |
|------|------|--------|---------|
| 双指针验证 | 左右指针同时向中间走 | O(n) / O(1) | 验证给定字符串是否是回文 |
| 中心扩展 | 从每个中心向两边扩 | O(n²) / O(1) | 找最长回文子串 |
| Manacher | 线性扫描 | O(n) / O(n) | 竞赛用，面试不要求 |

**中心扩展图解**：

```
"b a b a d"
     ↑
  中心 i=2

奇数扩展(中心=2)：b a [b] a d  → "bab"
偶数扩展(中心=2,3)：b a [b a] d → "aba"
```

### 2.5 前缀和 + 哈希表

> 🎯 **核心思想**：`sum(l..r) = prefix[r+1] - prefix[l]`

当题目问"有多少个子数组的和等于K"时，暴力是O(n²)遍历所有子数组。前缀和+哈希的优化思路是：

```python
# 暴力 O(n²)
for l in range(n):
    for r in range(l, n):
        if sum(nums[l:r+1]) == k:  # 每次O(n)
            ans += 1

# 前缀和优化 O(n)
# sum(l..r) = prefix[r+1] - prefix[l] = k
# → prefix[l] = prefix[r+1] - k
# 所以我们只需一边遍历，一边用哈希表记录见过的前缀和
```

## 三、五大核心套路模板

### 套路1：对撞双指针

**识别特征**：
- 有序数组（或可以先排序）
- 找两个/三个元素满足条件
- 要最大化/最小化某个值

**模板代码**（直接背诵！）：

```python
def two_pointer_template(nums):
    """对撞双指针万能模板"""
    nums.sort()  # 通常需要先排序
    n = len(nums)
    L, R = 0, n - 1
    
    while L < R:
        current = nums[L] + nums[R]  # 或 nums[L]*nums[R] 等其他运算
        
        if current == target:
            # 找到答案
            return [L, R]
        elif current < target:
            L += 1  # 需要更大 → 左指针右移
        else:
            R -= 1  # 需要更小 → 右指针左移
    
    return None  # 没找到
```

**适用范围**：两数之和、三数之和、四数之和、盛最多水的容器、最接近的三数之和

### 套路2：快慢双指针

**识别特征**：
- 原地修改数组
- 需要"过滤"某些元素
- 链表判环

**模板代码**（直接背诵！）：

```python
def fast_slow_template(nums):
    """快慢双指针万能模板 — 原地过滤"""
    slow = 0  # 慢指针：指向"应该放的位置"
    
    for fast in range(len(nums)):  # 快指针：探路
        if 满足条件(nums[fast]):
            nums[slow] = nums[fast]
            slow += 1
    
    return slow  # 有效长度
```

**适用范围**：移动零、删除有序数组重复项、移除指定元素、压缩字符串

### 套路3：可变滑动窗口

**识别特征**：
- 子串/子数组问题
- 要求"最长"、"最短"、"包含"
- 需要满足某种计数/字符条件

**模板代码**（直接背诵！）：

```python
from collections import defaultdict, Counter

def sliding_window_template(s):
    """可变滑动窗口万能模板"""
    window = defaultdict(int)  # 窗口内的字符/元素计数
    need = Counter(target)      # 需要的字符/元素
    L = 0
    ans = 0  # 或 float('inf') 取决于求最长还是最短
    
    for R in range(len(s)):
        # 1. 扩展窗口：把 s[R] 加入窗口
        window[s[R]] += 1
        
        # 2. 收缩窗口：当窗口满足条件时缩小
        while 窗口满足条件():
            # 更新答案
            ans = max/min(ans, R - L + 1)
            
            # 移出左边界
            window[s[L]] -= 1
            if window[s[L]] == 0:
                del window[s[L]]
            L += 1
    
    return ans
```

**适用范围**：无重复字符最长子串、最小覆盖子串、长度最小子数组、至多包含K个不同字符的最长子串

### 套路4：中心扩展法（回文）

**识别特征**：
- 回文子串问题
- 求"最长回文子串"、"回文子串个数"

**模板代码**（直接背诵！）：

```python
def palindrome_template(s):
    """中心扩展法万能模板"""
    def expand(left, right):
        """从中心(left, right)向两边扩展，返回最长回文子串"""
        while left >= 0 and right < len(s) and s[left] == s[right]:
            left -= 1
            right += 1
        # 退出时left和right已经越界/不相等，回文是(left+1, right-1]
        return s[left + 1:right]
    
    best = ""
    for i in range(len(s)):
        # 奇数长度回文：中心是单个字符
        odd = expand(i, i)
        # 偶数长度回文：中心是两个字符之间
        even = expand(i, i + 1)
        
        # 取更长的一个
        if len(odd) > len(best):
            best = odd
        if len(even) > len(best):
            best = even
    
    return best
```

**适用范围**：最长回文子串、回文子串个数、验证回文串II（删除一个字符后是否为回文）

### 套路5：前缀和 + 哈希表

**识别特征**：
- "子数组之和等于K"
- "连续子数组的个数"
- 涉及区间求和

**模板代码**（直接背诵！）：

```python
from collections import defaultdict

def prefix_sum_hash_template(nums, k):
    """前缀和+哈希表万能模板"""
    prefix_count = defaultdict(int)
    prefix_count[0] = 1  # 关键：前缀和为0出现1次（空数组）
    
    current_sum = 0
    ans = 0
    
    for x in nums:
        current_sum += x
        
        # sum(l..r) = k → prefix[r+1] - prefix[l] = k
        # → prefix[l] = prefix[r+1] - k
        need = current_sum - k
        ans += prefix_count[need]
        
        prefix_count[current_sum] += 1
    
    return ans
```

**适用范围**：和为K的子数组、和为K的子数组个数、路径总和III（树的前缀和）、连续数组（0和1个数相等的最长子数组）

### 题目1：三数之和 ⭐⭐⭐

📝 **题目**：给定整数数组 `nums`，找出所有和为0且不重复的三元组。

```
输入：nums = [-1, 0, 1, 2, -1, -4]
输出：[[-1, -1, 2], [-1, 0, 1]]
```

🧠 **解题思路**：

这道题是"两数之和"的升级版。核心思路是：**排序 + 固定一个数 + 对撞双指针**。

```
Step 1：排序数组
        [-4, -1, -1, 0, 1, 2]
         ↑
       固定 i=0（nums[0]=-4）

Step 2：在 i+1 到末尾中，用双指针找两数之和 = -(-4) = 4
        L=1(-1), R=5(2): -1+2=1 < 4 → L++
        L=2(-1), R=5(2): -1+2=1 < 4 → L++
        L=3(0), R=5(2): 0+2=2 < 4 → L++
        L=4(1), R=5(2): 1+2=3 < 4 → L++
        没找到，i++

Step 3：固定 i=1（nums[1]=-1，跳过重复的i=2）
        L=2(-1), R=5(2): -1+2=1 < 1 → 等...
        修正：目标是 -(-1) = 1
        L=2(-1), R=5(2): -1+2=1 ✅ = 1！找到 [-1, -1, 2]
        继续搜索...

Step 4：固定 i=3（nums[3]=0）
        L=4(1), R=5(2): 1+2=3 > 0 → R--
        L=4(1), R=4? → L=R 退出
```

📝 **完整代码**：

```python
def threeSum(nums: list[int]) -> list[list[int]]:
    """
    三数之和 — 排序 + 固定一数 + 对撞双指针
    时间复杂度 O(n²)，空间 O(log n)（排序栈空间）
    """
    nums.sort()                   # 核心1：排序是双指针的前提
    n = len(nums)
    res = []
    
    for i in range(n - 2):        # 固定第一个数，留两个位置给L和R
        # 核心2：跳过重复的固定值
        if i > 0 and nums[i] == nums[i - 1]:
            continue
        
        # 核心3：提前剪枝 — 最小的三个数都>0，不可能有解
        if nums[i] + nums[i + 1] + nums[i + 2] > 0:
            break
        
        # 核心4：提前剪枝 — 当前最大三个数<0，当前i无解
        if nums[i] + nums[n - 2] + nums[n - 1] < 0:
            continue
        
        L, R = i + 1, n - 1       # 对撞双指针
        while L < R:
            total = nums[i] + nums[L] + nums[R]
            
            if total == 0:
                res.append([nums[i], nums[L], nums[R]])
                L += 1
                R -= 1
                # 核心5：跳过内层重复值
                while L < R and nums[L] == nums[L - 1]:
                    L += 1
                while L < R and nums[R] == nums[R + 1]:
                    R -= 1
            elif total < 0:
                L += 1            # 和太小 → 左指针右移增大和
            else:
                R -= 1            # 和太大 → 右指针左移减小和
    
    return res
```

📊 **复杂度分析**：
- 时间：O(n²) — 外层O(n)，内层双指针O(n)
- 空间：O(log n) — 排序的递归栈（不计输出）

⚠️ **易错点**：
1. **去重是关键**：`i > 0 and nums[i] == nums[i-1]` 不是 `i > 0 and nums[i] == nums[i+1]`！前者跳过"已处理过的相同值"，后者会漏解
2. **内层去重时机**：找到解后再去重 `L` 和 `R`，不是在找到之前

### 题目2：无重复字符的最长子串 ⭐⭐⭐

📝 **题目**：给定字符串 `s`，找出其中不含有重复字符的最长子串的长度。

```
输入：s = "abcabcbb"
输出：3（子串 "abc"）
```

🧠 **解题思路**：

**滑动窗口 + 哈希表记录字符最近位置**。

```
s = "a b c a b c b b"
     ↑
    L=0, R=0: 窗口[a], seen={a:0}, ans=1

    L=0, R=1: 窗口[ab], seen={a:0, b:1}, ans=2

    L=0, R=2: 窗口[abc], seen={a:0, b:1, c:2}, ans=3

    L=0, R=3: ch='a', seen['a']=0 >= L=0 → 重复！
                L 跳到 seen['a']+1 = 1
                窗口变为[bca], seen['a']=3, ans=max(3, 3-1+1)=3

    L=1, R=4: ch='b', seen['b']=1 >= L=1 → 重复！
                L 跳到 seen['b']+1 = 2
                窗口变为[cab], seen['b']=4, ans=3

    依此类推...
    最终 ans=3
```

📝 **完整代码**：

```python
def lengthOfLongestSubstring(s: str) -> int:
    """
    无重复字符的最长子串 — 滑动窗口 + 哈希表
    时间复杂度 O(n)，空间 O(min(n, 128))
    """
    seen = {}            # 字符 → 最近出现的索引
    L = 0                # 窗口左边界
    ans = 0
    
    for R, ch in enumerate(s):   # R 是窗口右边界
        # 如果 ch 在当前窗口中已出现 → 收缩左边界
        if ch in seen and seen[ch] >= L:
            L = seen[ch] + 1     # 跳到重复字符的下一个位置
        
        seen[ch] = R             # 更新最近位置
        ans = max(ans, R - L + 1)  # 更新答案
    
    return ans
```

📊 **复杂度分析**：
- 时间：O(n) — 每个字符只被左右指针各遍历一次
- 空间：O(min(n, 128)) — 字符集大小（ASCII 128，Unicode可更多）

💡 **小技巧**：`seen[ch] >= L` 这个判断很精妙——只关心"在当前窗口内"的重复，历史位置的重复不管！

### 题目3：最长回文子串 ⭐⭐⭐

📝 **题目**：给定字符串 `s`，找出最长的回文子串。

```
输入：s = "babad"
输出："bab"（"aba" 也是正确答案）
```

🧠 **解题思路**：

**中心扩展法**——从每个位置出发，向两边扩展找最长回文。

```
s = "b a b a d"
索引 0 1 2 3 4

i=0：奇数扩展 → "b" (长1), 偶数扩展(i,i+1) → "" (ba不相等)
i=1：奇数扩展 → "a" → "bab" (长3), 偶数扩展 → "ba" 不等
i=2：奇数扩展 → "b" → "aba" → "babad"不等 → "aba" (长3), 偶数扩展 → "ab"不等
i=3：奇数扩展 → "a" → "bad"不等, 偶数扩展 → "ad"不等
i=4：奇数扩展 → "d"

最长："bab"(或"aba")，长度3
```

📝 **完整代码**：

```python
def longestPalindrome(s: str) -> str:
    """
    最长回文子串 — 中心扩展法
    时间复杂度 O(n²)，空间 O(1)
    """
    def expand(left: int, right: int) -> str:
        """从中心向两边扩展，返回最长回文子串"""
        while left >= 0 and right < len(s) and s[left] == s[right]:
            left -= 1
            right += 1
        # 退出时 s[left] != s[right] 或越界，回文在 (left, right) 之间
        return s[left + 1:right]
    
    best = ""
    for i in range(len(s)):
        odd = expand(i, i)       # 奇数长度回文，中心是 s[i]
        even = expand(i, i + 1)  # 偶数长度回文，中心在 s[i] 和 s[i+1] 之间
        
        if len(odd) > len(best):
            best = odd
        if len(even) > len(best):
            best = even
    
    return best
```

📊 **复杂度分析**：
- 时间：O(n²) — 每个中心最坏扩展O(n)
- 空间：O(1) — 只用了几个变量

💡 **小技巧**：面试时中心扩展法足够！Manacher算法（O(n)）虽然有O(n)的理论最优解，但代码太长太绕，面试很难一次写对。除非面试官明确问"能不能做到线性时间"，否则不需要。

### 题目4：和为 K 的子数组 ⭐⭐⭐

📝 **题目**：给定整数数组 `nums` 和整数 `k`，统计和为 `k` 的连续子数组的个数。

```
输入：nums = [1, 1, 1], k = 2
输出：2（子数组 [1,1] 和 [1,1]）
```

🧠 **手把手推演**：

```
nums = [1, 2, 3], k = 3
prefix_count = {0: 1}  ← 前缀和0出现了1次（空前缀）

i=0: current_sum = 1
     need = 1 - 3 = -2 → prefix_count中没有，ans += 0
     prefix_count[1] = 1 → {0:1, 1:1}

i=1: current_sum = 3
     need = 3 - 3 = 0 → prefix_count中有0(1次)，ans += 1 ✅ ✅ 找到了子数组[1,2]
     prefix_count[3] = 1 → {0:1, 1:1, 3:1}

i=2: current_sum = 6
     need = 6 - 3 = 3 → prefix_count中有3(1次)，ans += 1 ✅ ✅ 找到了子数组[3]
     prefix_count[6] = 1 → {0:1, 1:1, 3:1, 6:1}

ans = 2（子数组[1,2]和[3]）
```

📝 **完整代码**：

```python
from collections import defaultdict

def subarraySum(nums: list[int], k: int) -> int:
    """
    和为K的子数组 — 前缀和 + 哈希表
    时间复杂度 O(n)，空间 O(n)
    
    核心公式：sum(l..r) = prefix[r] - prefix[l-1] = k
    即：prefix[l-1] = prefix[r] - k
    遍历时，我们已知 prefix[r]（即 current_sum），
    检查哈希表中是否有 prefix[r] - k
    """
    prefix_count = defaultdict(int)
    prefix_count[0] = 1  # 空前缀，和为0，出现1次
    
    current_sum = 0
    ans = 0
    
    for x in nums:
        current_sum += x
        
        # 问：之前有多少个前缀和 = current_sum - k？
        # 那就对应了多少个以当前位置为右端点的子数组
        need = current_sum - k
        ans += prefix_count[need]
        
        # 当前前缀和入表（供后面的位置使用）
        prefix_count[current_sum] += 1
    
    return ans
```

📊 **复杂度分析**：
- 时间：O(n)
- 空间：O(n) — 哈希表

⚠️ **易错点**：必须先查 `need`，再加 `current_sum` 到哈希表。如果反过来，当 k=0 时会把空子数组也算进去！

### 题目5：最小覆盖子串 ⭐⭐⭐⭐

📝 **题目**：给定字符串 `s` 和 `t`，找出 `s` 中包含 `t` 所有字符的最小子串。如果不存在，返回空字符串。

```
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
```

🧠 **解题思路**：

这是滑动窗口的 Hard 巅峰题，核心技巧是用 **`missing` 计数器** 代替每次检查 Counter。

```
s = "ADOBECODEBANC", t = "ABC"
need = {'A':1, 'B':1, 'C':1}, missing = 3

R=0 'A': need['A']:1→0, missing:3→2  窗口：[A]
R=1 'D': 'D'不在need, 忽略           窗口：[AD]
R=2 'O': 'O'不在need, 忽略           窗口：[ADO]
R=3 'B': need['B']:1→0, missing:2→1  窗口：[ADOB]
R=4 'E': 'E'不在need, 忽略           窗口：[ADOBE]
R=5 'C': need['C']:1→0, missing:1→0  窗口：[ADOBEC] ← missing=0！

收缩：L=0 'A', need['A']:0→1, missing:0→1  窗口：[DOBEC] 不再满足
R=6 'O': 忽略                          窗口：[DOBECO]
...继续直到再次 missing=0...
R=9 'A': need['A']:1→0, missing:1→0  窗口：[DOBECODEBA]

收缩：
  L=1 'D': 忽略, 窗口长度=8, start=1
  L=2 'O': 忽略, 窗口长度=7, start=2
  L=3 'B': need['B']:0→1, missing:0→1  不再满足

R=10 'N': 忽略
R=11 'C': need['C']:0→-1, missing不动(=1), need['C']=-1表示多余
          ...需要继续等...

R=12 'B': need['B']:1→0, missing:1→0  missing=0!

收缩：
  记录 min_len=4 (窗口[ANC?]) → "BANC"，长度4
```

📝 **完整代码**：

```python
from collections import Counter

def minWindow(s: str, t: str) -> str:
    """
    最小覆盖子串 — 可变滑动窗口 + missing计数器
    时间复杂度 O(n)，空间 O(k)，k为字符集大小
    """
    need = Counter(t)       # 需要的字符及其数量
    missing = len(t)        # 还需要匹配的字符总数
    L = 0
    start, min_len = 0, float('inf')
    
    for R, ch in enumerate(s):
        # 扩展窗口
        if ch in need:
            if need[ch] > 0:       # 当前字符仍被需要
                missing -= 1
            need[ch] -= 1          # 用掉一个（可能变负，表示多余）
        
        # 收缩窗口 — 当所有字符都匹配
        while missing == 0:
            # 更新最短结果
            if R - L + 1 < min_len:
                start = L
                min_len = R - L + 1
            
            # 移出左边界
            left_ch = s[L]
            if left_ch in need:
                need[left_ch] += 1
                if need[left_ch] > 0:  # 移出后不够了
                    missing += 1
            L += 1
    
    return "" if min_len == float('inf') else s[start:start + min_len]
```

📊 **复杂度分析**：
- 时间：O(n) — 每个字符最多被左右指针各访问一次
- 空间：O(k) — Counter大小

💡 **核心技巧**：`missing` 计数器避免了每次需要 O(k) 扫描 need 来判断是否满足条件，从而把整体复杂度从 O(nk) 降到 O(n)。

## 五、性能优化与常见陷阱

### ⚠️ Python字符串操作的性能陷阱

| 操作 | ❌ 坏写法 | ✅ 好写法 | 为什么 |
|------|----------|----------|--------|
| 循环拼接 | `s += ch` | `"".join(chars)` | +每次创建新字符串 O(n²) |
| 频繁切片 | 循环中`s = s[1:]` | 用索引/双指针代替 | 每次切片O(n) |
| 字符存在性 | `ch in list(s)` | `ch in set(s)` 或 `ch in s` | 列表O(n)，集合/字符串O(1) |
| 计数 | 循环`s.count(ch)` | 一次Counter | 每次count是O(n) |
| 排序后去重 | 手动循环 | `set()` 或 `dict.fromkeys()` | 更简洁且O(n) |

### 🎯 双指针常见错误

```
❌ 错误1：对撞指针忘了排序
  nums = [3, 2, 4], target = 6
  直接用双指针 → 误判！

❌ 错误2：三数之和去重用错条件
  if nums[i] == nums[i+1]: continue  ← 错误！
  应该是：if i > 0 and nums[i] == nums[i-1]: continue

❌ 错误3：滑动窗口收缩条件写错
  while 满足条件:    ← 如果求"最短"，应while满足
  while 不满足条件:  ← 如果求"最长"，应while不满足
```

### 🚀 滑动窗口性能优化

```python
# ❌ 每次检查都重新统计
while 满足条件(s[L:R+1]):  # O(n²)
    ...

# ✅ 增量更新计数
need[ch] -= 1  # O(1)
missing -= 1   # O(1)
```

## 七、面试追问方向

### 💬 常见追问与回答

**小白问："三数之和能不用排序吗？"**
**答**：理论上可以用哈希表做（类似两数之和），但去重会非常麻烦。排序虽然多花 O(n log n) 时间，但让去重变得极其简单，是面试最优解。

**小白问："为什么最小覆盖子串用 missing 而不是每次检查 need？"**
**答**：如果每次收缩窗口都 O(k) 检查 need（k 是字符集大小），虽然不影响大 O（k 是常数），但在面试中能展示你对细节的追求。用 `missing` 让检查变成 O(1)，是一个亮眼的优化点。

**小白问："前缀和 + 哈希表能处理 k=0 的情况吗？"**
**答**：可以！`prefix_count[0] = 1` 就是为此设计的。当遇到子数组自身和为0时，就能找到。

**小白问："Manacher 算法（O(n) 回文）面试需要掌握吗？"**
**答**：不需要。中心扩展法 O(n²) 在面试中完全够用，且代码简洁不易错。除非面试官明确要求线性时间解法，否则中心扩展法是最佳选择。

**小白问："双指针和滑动窗口有什么区别？"**
**答**：滑动窗口是双指针的子集。双指针更通用（对撞、快慢、分离），滑动窗口特指"维护一个可变大小的窗口，左右指针同向移动"的场景。所有滑动窗口问题都可以用双指针解决，但双指针问题不都能用滑动窗口。

**小白问："OD 机考能用 Python 的 sorted() 吗？"**
**答**：完全可以！Python 内置函数都是允许的。而且 `sorted()` 和 `.sort()` 是 Timsort，性能极好。
