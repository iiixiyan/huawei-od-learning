## 📌 前言：为什么你需要学哈希集合？

想象你是一个图书管理员，面对一屋子的书。如果有人问你"《三体》这本书在不在书架上？"，最笨的办法是一本一本地翻——这就是**遍历查找**（O(n)）。但如果你的书架有编号系统，你知道"文学类→科幻→刘慈欣→三体"这个精确位置，你就能**瞬间**回答——这就是**哈希查找**（O(1)）。

**哈希集合（HashSet）** 就是这样一个"瞬间查找"的神器。它记不住顺序，但能记住"这个东西我见过没有"。在编程世界里，它有三大核心能力：

| 能力 | 生活类比 | 时间复杂度 |
|------|---------|-----------|
| **去重** | 刷脸进门时，系统记住谁进过 | O(1) |
| **判存在** | 查字典：这个词我有印象吗？ | O(1) |
| **记录状态** | 旅游打卡：这个景点我去过了 | O(1) |

**刷题核心套路**：当你遇到"判断重复""找丢失的""检测循环"这类问题时，第一反应就应该是——**我用 HashSet 能不能解决？**

## 🧱 第一章：哈希表基础——从底层理解到"人话"

### 1.1 哈希函数：神奇的"指纹生成器"

**哈希函数（Hash Function）** 就是把任意数据映射成一个固定大小数字的"加工厂"。比如：

```
"苹果" → 哈希函数 → 编号 42
"香蕉" → 哈希函数 → 编号 17
"橘子" → 哈希函数 → 编号 89
```

生活类比：每个人的身份证号就是"人"的哈希值——从名字、生日、籍贯这些信息计算出来的唯一编号。

### 1.2 哈希冲突：两个人身份证号相同怎么办？

好的哈希函数尽量让不同输入得到不同输出，但理论上总会有两个不同的输入得到相同的哈希值——这就叫**哈希冲突**。

解决方法有两种：
1. **链地址法**：在冲突的位置挂一个链表，把所有哈希值相同的元素串起来
2. **开放地址法**：如果位置被占了，就往后找空位

**对于刷题的你**，不需要自己实现，但要知道这些原理，因为你未来会听到"哈希碰撞攻击"这样的面试追问。

### 1.3 Python 中的 HashSet：set

Python 的 `set` 就是基于哈希表实现的集合：

```python
# 创建 set
visited = set()          # 空集合
numbers = {1, 2, 3}      # 带初始值

# 核心操作
visited.add(5)           # 添加元素 — O(1)
visited.remove(5)        # 删除元素 — O(1)
5 in visited             # 判断是否存在 — O(1)
len(visited)             # 集合大小 — O(1)

# 集合运算（基于哈希，效率极高）
a = {1, 2, 3}
b = {2, 3, 4}
a & b   # 交集 → {2, 3}
a | b   # 并集 → {1, 2, 3, 4}
a - b   # 差集 → {1}
```

**⚠️ 重要提醒**：`set` 是无序的。如果你需要保持插入顺序，可以用 Python 3.7+ 的 `dict`（它维护插入顺序）或者 `collections.OrderedDict`。

## 🎯 第二章：核心套路一——"哈希表查补数"（两数之和）

### 2.1 问题：什么是"两数之和"？

> 给你一个整数数组 `nums` 和一个目标值 `target`，请你在数组中找出**和为目标值**的两个整数，并返回它们的下标。

**示例**：
```
输入：nums = [2, 7, 11, 15], target = 9
输出：[0, 1]
解释：因为 nums[0] + nums[1] = 2 + 7 = 9
```

### 2.2 暴力解法（你能想到的第一个思路）

```python
def twoSum_bruteforce(nums, target):
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []
```

**问题**：两个嵌套循环，时间复杂度 O(n²)。当 n = 10^5 时，需要执行约 5×10^9 次操作——电脑得跑几十秒。

### 2.3 哈希表解法："存已遍历，查补数"

**核心思想**：遍历数组时，把每个数"记下来"。对于当前数 `num`，我们想知道"之前有没有一个数等于 `target - num`"——用哈希表查，O(1)。

**生活类比**：你在超市买了两样东西，总价正好100元。你拿起一个标价35元的商品，心里默算："另一个应该是65元。"然后你翻购物车——车里有之前拿的商品，你挨个看价格（暴力法）。更好的方法是：你每拿一个商品就在手机备忘录记下价格，然后拿起35元的商品时，查一下备忘录里有没有65元——O(1)查询。

**算法推演（step by step）**：

```
nums = [2, 7, 11, 15], target = 9
seen = {}  # 哈希表，key=数字, value=下标

第1步：i=0, num=2
  complement = 9 - 2 = 7
  7 在 seen 中吗？→ 不在
  seen[2] = 0  → seen = {2: 0}

第2步：i=1, num=7
  complement = 9 - 7 = 2
  2 在 seen 中吗？→ 在！seen[2] = 0
  返回 [0, 1] ✅
```

**代码模板**：

```python
def twoSum(nums: list[int], target: int) -> list[int]:
    seen = {}  # 哈希表：存"已经遍历过的数字→下标"
    for i, num in enumerate(nums):
        complement = target - num  # 计算"补数"
        if complement in seen:     # 查哈希表：补数出现过吗？
            return [seen[complement], i]  # 找到了！
        seen[num] = i  # 没找到，把当前数记下来
    return []
```

**复杂度分析**：
- 时间复杂度：O(n) —— 只遍历一次数组，每次操作 O(1)
- 空间复杂度：O(n) —— 最坏情况存储 n 个元素

### 2.4 常见坑和面试追问

**坑1**：不能用同一个元素两次。我们的解法自然避免了——因为先查哈希表再添加，所以不会把自己算进去。

**坑2**：如果有重复元素怎么处理？题目说"每个输入只有唯一答案"，所以不用担心。但如果允许重复答案，我们需要用 `collections.defaultdict(list)` 存所有下标。

**追问1**：如果数组是排序好的，你能用双指针吗？
答：可以！排序后使用左右双指针，时间复杂度 O(n) 且空间复杂度 O(1)。但哈希表解法更通用（不要求排序）。

**追问2**：如果找三个数之和（三数之和）呢？
答：排序 + 双指针，或者固定一个数后变成"两数之和"问题。时间复杂度 O(n²)。

## 🎯 第三章：核心套路二——"set检测循环"（快乐数）

### 3.1 问题：什么是快乐数？

> 对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。然后重复这个过程。如果最终能变成 1，就是快乐数；如果进入无限循环，就不是。

**示例**：判断 19 是不是快乐数
```
1² + 9² = 1 + 81 = 82
8² + 2² = 64 + 4 = 68
6² + 8² = 36 + 64 = 100
1² + 0² + 0² = 1  → 到达 1，快乐数！🎉
```

**示例**：判断 2 是不是快乐数
```
2² = 4
4² = 16
1² + 6² = 37
3² + 7² = 58
5² + 8² = 89
8² + 9² = 145
1² + 4² + 5² = 42
4² + 2² = 20
2² + 0² = 4   ← 又回到4了！进入循环，不是快乐数 ❌
```

### 3.2 核心思路：Set检测循环

**生活类比**：你走进一个迷宫，边走边在墙上做记号。如果你走到一个已经做过记号的岔路口，就知道自己在绕圈子——这就是 set 检测循环。

**算法推演**：

```
n = 19, seen = {}

第1轮：n=19，不在seen中 → seen.add(19)
  计算下一位：get_next(19) = 1²+9² = 82

第2轮：n=82，不在seen中 → seen.add(82)
  计算下一位：get_next(82) = 8²+2² = 68

第3轮：n=68，不在seen中 → seen.add(68)
  计算下一位：get_next(68) = 6²+8² = 100

第4轮：n=100，不在seen中 → seen.add(100)
  计算下一位：get_next(100) = 1²+0²+0² = 1

第5轮：n=1 → n == 1，返回 True ✅
```

**不快乐数的推演**（以 2 为例）：

```
n=2, seen = {}

第1轮：n=2, 不在seen → seen={2}, n=4
第2轮：n=4, 不在seen → seen={2,4}, n=16
第3轮：n=16, 不在seen → seen={2,4,16}, n=37
第4轮：n=37, 不在seen → seen={2,4,16,37}, n=58
...
第8轮：n=4, 4已经在seen中 → 检测到循环，返回 False ✅
```

### 3.3 代码实现

```python
def isHappy(n: int) -> bool:
    def get_next(num: int) -> int:
        """计算每个位置数字的平方和"""
        total = 0
        while num > 0:
            digit = num % 10          # 取最后一位数字
            total += digit * digit    # 平方累加
            num //= 10                # 去掉最后一位
        return total
    
    seen = set()          # 记录已经出现过的数字
    while n != 1 and n not in seen:
        seen.add(n)       # 标记当前数字已出现
        n = get_next(n)   # 计算下一个数字
    return n == 1         # 最终判断是不是到了1
```

**代码逐行解读**：
- `get_next(19)`：先取 19%10=9，total+=81，19//10=1；再取 1%10=1，total+=1，1//10=0，结束。返回 82。✅
- `seen = set()`：起"记账本"的作用
- `while n != 1 and n not in seen`：只要没到1且没发现循环，就继续
- `return n == 1`：如果跳出循环时 n 恰好是 1，返回 True；否则（检测到循环）返回 False

### 3.4 进阶解法：快慢指针（节省空间）

除了用 HashSet，还可以用**快慢指针**检测循环——就像在环形跑道上，跑得快的人最终会追上跑得慢的人。

```python
def isHappy_fast_slow(n: int) -> bool:
    def get_next(num: int) -> int:
        total = 0
        while num > 0:
            digit = num % 10
            total += digit * digit
            num //= 10
        return total
    
    slow = n
    fast = get_next(n)
    while fast != 1 and slow != fast:
        slow = get_next(slow)         # 慢指针走1步
        fast = get_next(get_next(fast))  # 快指针走2步
    return fast == 1
```

**优点**：空间复杂度 O(1)，不需要额外 set。

### 3.5 复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| HashSet | O(log n) | O(log n) |
| 快慢指针 | O(log n) | O(1) |

注：为什么是 O(log n)？因为每次 `get_next` 操作处理的位数大约为 log₁₀(n)，且循环次数也是常数级别的（最多几百次）。

## 🎯 第四章：核心套路三——"只从序列起点开始"（最长连续序列）

### 4.1 问题

> 给定一个未排序的整数数组 nums，找出数字连续的最长序列的长度。要求 O(n) 时间复杂度。

**示例**：
```
输入：[100, 4, 200, 1, 3, 2]
输出：4
解释：最长连续序列是 [1, 2, 3, 4]，长度为 4
```

### 4.2 为什么不能排序？

你可能会想："排序不就行了？排序后遍历找连续段，O(n log n)。"

但题目明确要求**O(n)**。排序本身至少是 O(n log n)，所以这条路走不通。

### 4.3 核心优化思想："只从起点开始"

**暴力思路**：对每个数字，看它+1在不在数组中，在就继续找。但这样会重复计算——比如数组有 [1,2,3,4]，从 1 开始数出长度 4，从 2 又开始数出长度 3...

**优化**：**只从序列的起点（即 num-1 不在数组中）开始计数**。

**生活类比**：你在爬一座连绵的山脉。如果你从山脚（起点）开始爬，你能爬到山顶。但如果你从半山腰开始爬，你也能到山顶，但你已经走了一段"重复的路"。聪明的做法是：只从山脚出发的人开始计数。

**算法推演**：

```
nums = [100, 4, 200, 1, 3, 2]
num_set = {1, 2, 3, 4, 100, 200}

遍历 set（注意不是遍历原数组，避免重复数字干扰）：

检查 100：100-1=99 在 set 中吗？→ 不在 → 100 是起点！
  从 100 开始：101 在 set 中吗？→ 不在 → 长度=1
  longest = max(0, 1) = 1

检查 4：4-1=3 在 set 中吗？→ 在 → 4 不是起点，跳过！

检查 200：200-1=199 在 set 中吗？→ 不在 → 200 是起点！
  从 200 开始：201 在 set 中吗？→ 不在 → 长度=1
  longest = max(1, 1) = 1

检查 1：1-1=0 在 set 中吗？→ 不在 → 1 是起点！🎯
  从 1 开始：2 在 set 中 → 长度=2
           3 在 set 中 → 长度=3
           4 在 set 中 → 长度=4
           5 在 set 中 → 长度=5 → 不对，5不在set中，停止
  longest = max(1, 4) = 4

检查 2：2-1=1 在 set 中 → 跳过
检查 3：3-1=2 在 set 中 → 跳过

结果：longest = 4 ✅
```

**为什么每个元素只被访问两次？**
- 外层循环遍历 set，检查 `num-1` 是否在 set 中——O(1)
- 内层循环只对**起点**元素执行，向后计数
- 被跳过的元素（非起点）不被内层循环访问
- 每个元素最多成为一次"起点"的内层循环的一部分

所以总时间复杂度 = O(n) + O(n) = O(n)。

### 4.4 代码实现

```python
def longestConsecutive(nums: list[int]) -> int:
    num_set = set(nums)      # 去重 + 转为哈希集合
    longest = 0
    
    for num in num_set:      # 遍历集合（注意不是原数组）
        # 关键判断：只有 num-1 不在 set 中，才说明 num 是起点
        if num - 1 not in num_set:
            cur = num
            length = 1
            # 向后计数：cur+1, cur+2, ... 直到不在 set 中
            while cur + 1 in num_set:
                cur += 1
                length += 1
            longest = max(longest, length)
    
    return longest
```

### 4.5 常见坑

**坑1**：为什么遍历 `num_set` 而不是 `nums`？
答：因为原数组可能有重复元素（如 [1,0,1,2]），重复会干扰判断。遍历 set 天然去重。

**坑2**：为什么判断 `num - 1 not in num_set` 就够了？
答：这确保了每个连续序列只从它的最小值开始被计数一次。序列 [1,2,3,4] 中，只有 1 满足 0 不在 set 中，2、3、4 都不满足，所以它们不会触发内层循环。

### 4.6 面试追问

**追问**：如果允许 O(n log n)，排序法怎么写？
```python
def longestConsecutive_sort(nums):
    if not nums: return 0
    nums.sort()
    max_len = 1
    cur_len = 1
    for i in range(1, len(nums)):
        if nums[i] == nums[i-1]:
            continue  # 跳过重复
        if nums[i] == nums[i-1] + 1:
            cur_len += 1  # 连续
        else:
            max_len = max(max_len, cur_len)
            cur_len = 1
    return max(max_len, cur_len)
```
面试官可能会问你两种方法的优劣对比，记住：**哈希表法 O(n) 时间 O(n) 空间，排序法 O(n log n) 时间 O(1) 空间**。

## 🎯 第五章：核心套路四——"滑动窗口 + Set"（存在重复元素 II）

### 5.1 问题

> 给你一个整数数组 nums 和一个整数 k，判断数组中是否存在两个不同的索引 i 和 j，使得 `nums[i] == nums[j]` 且 `abs(i - j) <= k`。

**示例**：
```
nums = [1,2,3,1], k = 3 → true（两个1相距3）
nums = [1,0,1,1], k = 1 → true（两个1相距1）
nums = [1,2,3,1,2,3], k = 2 → false（相同元素最近距离 > 2）
```

### 5.2 核心思路：维护一个大小为 k 的滑动窗口

**生活类比**：你站在一个移动的传送带前，查看最近经过的 k 件商品里有没有重样的。传送带每移动一步，新的商品进入视野，最远的那件商品就离开视野——这就是"滑动窗口"。

**为什么是滑动窗口？**
因为条件要求 `abs(i - j) <= k`，所以对于当前索引 i，我们只需要关注它前面最多 k 个元素。超过 k 的元素已经"过期"了，可以丢弃。

### 5.3 算法推演

```
nums = [1, 2, 3, 1], k = 3
window = set()  # 窗口内的元素集合

i=0, num=1:
  1 在 window 中？→ 不在
  window.add(1) → window = {1}
  window 大小=1，不超过 k+1=4，不删除

i=1, num=2:
  2 在 window 中？→ 不在
  window.add(2) → window = {1, 2}
  window 大小=2，不超过4

i=2, num=3:
  3 在 window 中？→ 不在
  window.add(3) → window = {1, 2, 3}
  window 大小=3，不超过4

i=3, num=1:
  1 在 window 中？→ 在！返回 True ✅
```

**另一个例子**：
```
nums = [1, 2, 3, 1, 2, 3], k = 2
window = set()

i=0, num=1: 1不在→add→{1}, len=1≤3
i=1, num=2: 2不在→add→{1,2}, len=2≤3
i=2, num=3: 3不在→add→{1,2,3}, len=3≤3
i=3, num=1: 1在window中→返回 True?
  等等！仔细看：window当前有{1,2,3}
  1确实在window中，但这个1是nums[0]的1
  距离 = 3 - 0 = 3 > k = 2 ❌
```

**⚠️ 这里有个大坑！** 上面这个推演是错的。正确的做法是：**先检查是否有重复，再删除超出窗口范围的元素**。但上面的推演中，窗口大小正好是3=k+1，包含 nums[0]=1、nums[1]=2、nums[2]=3。当 i=3 时，window 里有 1（来自 nums[0]），但 nums[0] 已经超出了窗口范围（距离=3>2）！

所以正确的顺序应该是：
1. 先删除超出窗口的元素（如果窗口已满）
2. 再检查当前元素是否在窗口中
3. 把当前元素加入窗口

### 5.4 正确的算法推演

```
nums = [1, 2, 3, 1, 2, 3], k = 2
window = set()

i=0, num=1:
  先检查要删除的元素：i-k-1 = -3，无效，不删
  1 在 window 中？→ 不在
  window.add(1) → {1}

i=1, num=2:
  要删除的元素索引：1-2-1 = -2，无效
  2 在 window 中？→ 不在
  window.add(2) → {1, 2}

i=2, num=3:
  要删除的元素索引：2-2-1 = -1，无效
  3 在 window 中？→ 不在
  window.add(3) → {1, 2, 3}

i=3, num=1:
  要删除的元素索引：3-2-1 = 0，删除 nums[0]=1
  window.remove(1) → {2, 3}
  1 在 window 中？→ 不在 ✅
  window.add(1) → {2, 3, 1}

i=4, num=2:
  要删除的元素索引：4-2-1 = 1，删除 nums[1]=2
  window.remove(2) → {3, 1}
  2 在 window 中？→ 不在 ✅
  window.add(2) → {3, 1, 2}

i=5, num=3:
  要删除的元素索引：5-2-1 = 2，删除 nums[2]=3
  window.remove(3) → {1, 2}
  3 在 window 中？→ 不在 ✅
  window.add(3) → {1, 2, 3}

遍历结束，返回 False ✅
```

### 5.5 代码实现（更易理解的版本）

```python
def containsNearbyDuplicate(nums: list[int], k: int) -> bool:
    window = set()
    for i, num in enumerate(nums):
        # 步骤1：窗口满了，移除最左边的元素
        # 窗口大小 = i - (i-k-1) = k+1，所以当 i > k 时，需要移除 nums[i-k-1]
        if i > k:
            window.remove(nums[i - k - 1])
        
        # 步骤2：检查当前元素是否已在窗口中
        if num in window:
            return True
        
        # 步骤3：将当前元素加入窗口
        window.add(num)
    
    return False
```

**更简洁的写法**（通过 window 大小控制）：

```python
def containsNearbyDuplicate(nums: list[int], k: int) -> bool:
    window = set()
    for i, num in enumerate(nums):
        if num in window:
            return True
        window.add(num)
        # 当窗口大小超过 k 时，移除最左边的元素
        if len(window) > k:
            window.remove(nums[i - k])
    return False
```

### 5.6 知识点总结：滑动窗口

滑动窗口是解决"子数组/子字符串"问题的利器。核心三要素：
1. **窗口是什么**：一个连续的区间，由 left 和 right 指针界定
2. **什么时候扩大**：right 右移，加入新元素
3. **什么时候缩小**：left 右移，移除旧元素

本题中的固定大小滑动窗口是滑动窗口中最简单的一种。

## 🎯 第六章：补数配对练习（赎金信）

### 6.1 问题

> 给你两个字符串 ransomNote 和 magazine，判断 ransomNote 能否由 magazine 里面的字符构成。magazine 中的每个字符只能在 ransomNote 中使用一次。

**示例**：
```
ransomNote = "aa", magazine = "aab" → true
ransomNote = "aa", magazine = "ab" → false
```

### 6.2 核心思路：计数 + 减法

**生活类比**：你要做一盘菜（ransomNote），需要从冰箱（magazine）里拿食材。你需要检查冰箱里每种食材的数量是否够用。

**思路**：
1. 先统计冰箱（magazine）里每种食材有多少
2. 遍历菜品需要的食材，每用一个就消耗一个
3. 如果某种食材不够用了，返回 False

```python
from collections import Counter

def canConstruct(ransomNote: str, magazine: str) -> bool:
    # 统计 magazine 中每个字符的数量
    counter = Counter(magazine)
    
    # 遍历 ransomNote 中的每个字符
    for ch in ransomNote:
        # 如果这个字符不够用了，返回 False
        if counter[ch] == 0:
            return False
        # 消耗一个字符
        counter[ch] -= 1
    
    return True
```

### 6.3 复杂度分析

- 时间复杂度：O(m + n)，其中 m = len(magazine)，n = len(ransomNote)
- 空间复杂度：O(1) —— 因为只有26个小写字母，Counter 大小恒定

## 📚 今日总结

### 4个必须刻在脑子里的核心套路

1. **哈希表查补数**（两数之和）→ 空间换时间，O(n²)变O(n)
2. **Set检测循环**（快乐数）→ 记录已访问状态，检测重复
3. **只从起点开始**（最长连续序列）→ 避免重复计算，每个元素最多访问两次
4. **滑动窗口+Set**（存在重复元素II）→ 固定大小窗口 + 去重查询
