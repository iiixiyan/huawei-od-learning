## 一、为什么数组需要数学思维？🤔

### 🏠 生活例子：分摊账单 💰

你和3个朋友去吃饭，总共消费200元。现在要算**每个人自己点菜花了多少钱**。

```
菜单：  你      A      B      C
点菜：  25元   40元   35元   50元
小计：  25     65     100    150
总计：  200元
```

要算A除了自己之外其他人花了多少，就是：200 - 40 = 160元。

但如果换成乘法呢？**"除自身以外数组的乘积"** 就像是：算每个人的"除自己之外所有人的乘积"：

```
数组：     [2,  3,  4,  5]
自身以外积：[60, 40, 30, 24]
解释：     3×4×5  2×4×5  2×3×5  2×3×4
```

暴力法是每次乘一遍（O(n²)），但用**前缀积+后缀积**可以O(n)解决！

Day1 我们处理"静态移除"（快慢指针），Day2 处理"状态决策"（贪心/状态机），Day3 处理"数学关系"——利用数组元素之间的数学关系，通过巧妙的遍历策略解决问题。

### ⚡ 套路一：前缀积 + 后缀积

#### 📖 数学概念回顾

**前缀积**：从左边开始累乘到当前位置的积。
**后缀积**：从右边开始累乘到当前位置的积。

```
数组：     [2,  3,  4,  5]
前缀积：   [2,  6,  24, 120]   前缀积[i] = nums[0]×nums[1]×...×nums[i]
后缀积：   [120,60, 20, 5]     后缀积[i] = nums[i]×nums[i+1]×...×nums[n-1]
```

**除自身以外数组的乘积[i] = 前缀积[i-1] × 后缀积[i+1]**

```
答案[0] = 前缀积[-1]不存在 → 1 × 后缀积[1]=60 = 60
答案[1] = 前缀积[0]=2 × 后缀积[2]=20 = 40
答案[2] = 前缀积[1]=6 × 后缀积[3]=5 = 30
答案[3] = 前缀积[2]=24 × 后缀积[4]不存在 → 24 × 1 = 24
```

#### 生活例子：接力跑 🏃

想象一个接力赛，每棒选手跑一段距离。想知道**某个选手之外**的总距离：

```
选手：   A     B     C     D     E
距离：   100   200   150   180   120

想知道C之外的总距离：
方法1：每次重新算（暴力法）→ 100+200+180+120 = 600
方法2：先算前缀和 + 后缀和
  前缀和：A的总距离=100，AB总=300，ABC总=450...
  后缀和：E的总=120，DE总=300，CDE总=450...
  C之外 = 前缀到B的总和 + 后缀从D开始的总和
        = 300 + 300 = 600 ✓
```

#### 代码模板

```python
def productExceptSelf(nums: list[int]) -> list[int]:
    n = len(nums)
    answer = [1] * n  # 初始化为1（乘法的单位元）

    # 第一次遍历：计算左侧累积积
    left_product = 1
    for i in range(n):
        answer[i] = left_product  # 先把左边的积给answer[i]
        left_product *= nums[i]   # 更新左侧积

    # 第二次遍历：乘上右侧累积积
    right_product = 1
    for i in range(n - 1, -1, -1):
        answer[i] *= right_product  # 乘上右边的积
        right_product *= nums[i]   # 更新右侧积

    return answer
```

#### 小白口诀 🎵

> **从左到右走一遍，左侧累积存起来。从右到左再一遍，乘上右侧就出来。**

### ⚡ 套路二：左右两次遍历

#### 核心思想

很多数组问题需要同时利用"左侧信息"和"右侧信息"。解决策略是：
1. **从左往右遍历一次**，用左侧信息更新数组
2. **从右往左遍历一次**，用右侧信息更新数组
3. 合并两次遍历的结果

这个套路在**接雨水**、**加油站**、**除自身乘积**等题目中都有体现。

#### 生活例子：两面镜子 🪞

你站在两面镜子中间：

```
左镜 ← 你 → 右镜
```

从左镜你能看到你的左侧特征，从右镜你能看到右侧特征。只有把两面镜子的信息结合起来，才能看到"完整"的自己。

#### 模板

```python
def two_pass_template(nums):
    n = len(nums)
    result = [0] * n

    # 从左到右
    left_val = 初始值
    for i in range(n):
        result[i] = left_val  # 用左侧信息更新
        left_val = 根据规则更新(nums[i])

    # 从右到左（融合）
    right_val = 初始值
    for i in range(n - 1, -1, -1):
        result[i] = 融合(result[i], right_val)
        right_val = 根据规则更新(nums[i])

    return result
```

#### 小白口诀 🎵

> **从左到右记左边，从右到左记右边。两次遍历合一用，左右夹击无盲点。**

### ⚡ 套路三：双指针接雨水

#### 这是接雨水的专属套路

不同于Day1的"快慢指针"（写/读指针），接雨水的双指针是**从两端往中间走**，边走边计算能接多少雨水。

#### 核心思想

能接多少雨水，取决于**左右两边最高的柱子中较矮的那根**。

```
    |
  | | |
  | | | |
————————
```

位置i能接的水 = min(左边最高柱, 右边最高柱) - 自己的高度

如果这个值 > 0，就能接住；否则接不住。

#### 代码模板

```python
def trap(height: list[int]) -> int:
    if not height:
        return 0

    left, right = 0, len(height) - 1
    left_max = right_max = 0  # 左右两边遇到过的最高柱子
    water = 0

    while left < right:
        if height[left] < height[right]:
            # 左边矮，左边决定水位
            if height[left] >= left_max:
                left_max = height[left]  # 更新左边最高
            else:
                water += left_max - height[left]  # 接水
            left += 1
        else:
            # 右边矮（或相等），右边决定水位
            if height[right] >= right_max:
                right_max = height[right]  # 更新右边最高
            else:
                water += right_max - height[right]  # 接水
            right -= 1

    return water
```

#### 小白口诀 🎵

> **左右指针往中走，哪边更矮哪边流。遇到更高就更新，遇到更矮就接水。**

### 📝 第1题：除自身以外数组的乘积

**题目**：给你一个整数数组 `nums`，返回数组 `answer`，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余各元素的乘积。题目保证数组nums中所有元素的乘积在32位整数范围内。请不要使用除法，在 O(n) 时间复杂度内完成。

**难度**：⭐⭐ 中等

**示例**：
```
输入：nums = [1, 2, 3, 4]
输出：[24, 12, 8, 6]

输入：nums = [-1, 1, 0, -3, 3]
输出：[0, 0, 9, 0, 0]
```

#### 解题思路

**核心：不能用除法！**（如果可以用除法，直接算总乘积再除以每个元素就行）

**前缀积 + 后缀积法**：

```
nums    = [1,      2,      3,      4]
左累积  = [1,      1×1=1, 1×2=2, 1×2×3=6]
右累积  = [2×3×4=24, 3×4=12, 4,    1]
答案    = [1×24=24, 1×12=12, 2×4=8, 6×1=6]
```

**关键优化**：不存两个数组，用一个数组 + 一个变量。

#### 手把手推演

```
nums = [1, 2, 3, 4]

第1遍（从左到右，存左侧积到answer）：
  answer = [1, 1, 1, 1]
  left = 1

  i=0: answer[0] = left = 1, left *= 1 = 1
  i=1: answer[1] = left = 1, left *= 2 = 2
  i=2: answer[2] = left = 2, left *= 3 = 6
  i=3: answer[3] = left = 6, left *= 4 = 24

  现在 answer = [1, 1, 2, 6]  （每个位置存的是左侧所有元素的积）

第2遍（从右到左，乘右侧积）：
  right = 1

  i=3: answer[3] *= right = 6*1 = 6,  right *= 4 = 4
  i=2: answer[2] *= right = 2*4 = 8,  right *= 3 = 12
  i=1: answer[1] *= right = 1*12 = 12, right *= 2 = 24
  i=0: answer[0] *= right = 1*24 = 24, right *= 1 = 24

  最终 answer = [24, 12, 8, 6] ✓
```

#### 图解

```
原始数组:  [1,    2,    3,    4]

第1遍（左侧积）:
  answer: [1,    1,    2,    6]
           ↑     ↑     ↑     ↑
          左=1  左=1  左=2  左=6
          (无)  (1)  (1×2) (1×2×3)

第2遍（乘右侧积）:
  answer: [1×24 1×12  2×4  6×1]
           ↓     ↓     ↓     ↓
           24    12     8     6
          右=24 右=12  右=4  右=1
          (2×3×4) (3×4)  (4)  (无)
```

#### 代码实现

```python
def productExceptSelf(nums: list[int]) -> list[int]:
    n = len(nums)
    answer = [1] * n

    # 第一次遍历：左侧累积积
    left = 1
    for i in range(n):
        answer[i] = left
        left *= nums[i]

    # 第二次遍历：乘上右侧累积积
    right = 1
    for i in range(n - 1, -1, -1):
        answer[i] *= right
        right *= nums[i]

    return answer
```

#### 复杂度分析

- **时间复杂度**：O(n)，两次遍历
- **空间复杂度**：O(1)（输出数组不算额外空间）

#### 面试追问方向

**Q：如果可以用除法呢？怎么处理0？**
A：统计0的个数：
- 0个0：总乘积 ÷ 每个元素
- 1个0：只有0的位置是总乘积（其他元素乘积），其他位置是0
- ≥2个0：所有位置都是0

```python
def productExceptSelf(nums):
    n = len(nums)
    zero_count = nums.count(0)
    total_product = 1

    if zero_count >= 2:
        return [0] * n

    for x in nums:
        if x != 0:
            total_product *= x

    answer = []
    for x in nums:
        if zero_count == 1:
            answer.append(total_product if x == 0 else 0)
        else:
            answer.append(total_product // x)
    return answer
```

**Q：如果数组很大（比如10^5），前缀积溢出怎么办？**
A：可以用对数把乘法变成加法，但面试时一般不这么考。OD中通常保证在32位整数范围内。

### 📝 第2题：加油站

**题目**：在一条环路上有 `n` 个加油站，其中第 `i` 个加油站有汽油 `gas[i]` 升。你有一辆油箱容量无限的汽车，从第 `i` 个加油站开往第 `i+1` 个加油站需要消耗汽油 `cost[i]` 升。你从某个加油站出发，开始时油箱为空。如果你可以绕环路行驶一周，则返回出发时加油站的编号，否则返回 `-1`。

**难度**：⭐⭐ 中等

**示例**：
```
输入：gas = [1, 2, 3, 4, 5], cost = [3, 4, 5, 1, 2]
输出：3
解释：
  从3号加油站（gas[3]=4）出发：
  到4号：油量=4-1+5=8
  到0号：8-2+1=7  （注意是环路）
  到1号：7-3+2=6
  到2号：6-4+3=5
  回到3号：5-5+4=4
  成功！

输入：gas = [2, 3, 4], cost = [3, 4, 3]
输出：-1
解释：总油量9，总消耗10，不可能跑完全程。
```

#### 解题思路

**核心观察**：
1. 如果总油量 < 总消耗 → 不可能跑完 → 返回 -1
2. 如果总油量 ≥ 总消耗 → 一定存在一个起点

**贪心策略**：从第0个加油站开始尝试，记录当前油量 `tank`。如果出现 `tank < 0`（到不了下一站），说明从开始的这个起点到当前位置之间的任何一个加油站都不能作为起点。把起点设为 `i+1`，重置 `tank = 0`。

**为什么？** 因为如果从A出发到B油量<0，那么从A到B之间的任何一点C出发，到B时油量也会<0（因为从C到B能获得的油量不会比从A到B多）。

#### 手把手推演

```
gas  = [1, 2, 3, 4, 5]
cost = [3, 4, 5, 1, 2]

首先检查总油量：1+2+3+4+5=15
总消耗：3+4+5+1+2=15
15 ≥ 15 → 有解

开始尝试：
start=0, tank=0

i=0: gas[0]=1, cost[0]=3
  tank = 0 + 1 - 3 = -2
  tank < 0 → start=1, tank=0

i=1: gas[1]=2, cost[1]=4
  tank = 0 + 2 - 4 = -2
  tank < 0 → start=2, tank=0

i=2: gas[2]=3, cost[2]=5
  tank = 0 + 3 - 5 = -2
  tank < 0 → start=3, tank=0

i=3: gas[3]=4, cost[3]=1
  tank = 0 + 4 - 1 = 3
  tank > 0 → 继续

i=4: gas[4]=5, cost[4]=2
  tank = 3 + 5 - 2 = 6
  tank > 0 → 继续

遍历完，返回 start=3 ✓
```

#### 图解

```
路线图（环形）：
        0(1)
      /     \
    5(5)     1(2)
    |   ⭕   |
    4(4)     2(3)
      \     /
        3(4)

油耗箭头：0→1(3), 1→2(4), 2→3(5), 3→4(1), 4→0(2)

从起点3出发的油量变化：
位置3：油量=4
位置4：油量=4-1+5=8
位置0（绕回）：油量=8-2+1=7
位置1：油量=7-3+2=6
位置2：油量=6-4+3=5
回到位置3：油量=5-5+4=4
全程油量从未低于0 ✓
```

#### 代码实现

```python
def canCompleteCircuit(gas: list[int], cost: list[int]) -> int:
    total_gas = sum(gas)
    total_cost = sum(cost)

    if total_gas < total_cost:  # 总油量不够
        return -1

    start = 0
    tank = 0

    for i in range(len(gas)):
        tank += gas[i] - cost[i]  # 到下一站的净油量
        if tank < 0:              # 到不了了
            start = i + 1         # 尝试下一个起点
            tank = 0              # 重置油量

    return start
```

#### 复杂度分析

- **时间复杂度**：O(n)，一次遍历
- **空间复杂度**：O(1)

#### 面试追问方向

**Q：为什么 tank < 0 时，起点可以直接跳到 i+1？**
A：因为如果从 start 到 i 的累积净油量为负，那么从 start+1 到 i 的累积净油量只会更小（因为少拿了起点站的油），所以 start..i 之间的点都不能作为起点。

**Q：如果不需要提前判断总油量 ≥ 总消耗？**
A：可以先计算一次确定有解，也可以在遍历结束后再判断：

```python
def canCompleteCircuit(gas, cost):
    start = tank = total = 0
    for i in range(len(gas)):
        net = gas[i] - cost[i]
        tank += net
        total += net
        if tank < 0:
            start = i + 1
            tank = 0
    return start if total >= 0 else -1
```

这样只遍历一次。

### 📝 第3题：接雨水

**题目**：给定 `n` 个非负整数表示每个宽度为 1 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

**难度**：⭐⭐⭐⭐ 困难（但掌握套路就不难）

**示例**：
```
输入：height = [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
输出：6

示意图：
      |
    | | |
  | | | | | |
  ↓雨水积在凹槽里
```

#### 解题思路

**三种方法**：

**方法一：按列求（暴力法）**
对于每一列，找到左边最高和右边最高，取较矮的减去当前高度。
时间复杂度：O(n²)

**方法二：动态规划（提前算左右最高）**
先遍历两次，算出每个位置的左边最高和右边最高。
时间复杂度：O(n)，空间复杂度：O(n)

**方法三：双指针（最优）**
从两端往中间走，哪边矮就处理哪边。
时间复杂度：O(n)，空间复杂度：O(1)

#### 双指针法手把手推演

```
height = [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]

初始：left=0, right=11
     left_max=0, right_max=0
     water=0

第1步：height[0]=0 < height[11]=1
  左边矮，处理左边
  0 ≥ left_max(0) → left_max=0（没变）
  water += 0-0 = 0
  left=1

第2步：height[1]=1 < height[11]=1？不，1≥1
  右边处理（左右相等时处理哪边都可以，这里右边）
  height[11]=1 ≥ right_max(0) → right_max=1
  water += ...（右边）
  right=10

第3步：height[1]=1 vs height[10]=2
  height[1]=1 < 2 → 左边矮
  1 ≥ left_max(0) → left_max=1
  water += ...（左边）
  left=2

第4步：height[2]=0 vs height[10]=2
  0 < 2 → 左边矮
  0 < left_max(1) → water += 1-0 = 1（接住1单位水！）
  left=3, water=1

第5步：height[3]=2 vs height[10]=2
  2 == 2（处理右边）
  2 ≥ right_max(1) → right_max=2
  right=9

第6步：height[3]=2 vs height[9]=1
  1 < 2 → 右边矮
  1 < right_max(2) → water += 2-1 = 1
  right=8, water=2

第7步：height[3]=2 vs height[8]=2
  2 == 2（处理右边）
  2 ≥ right_max(2) → right_max=2
  right=7, water=2

第8步：height[3]=2 vs height[7]=3
  2 < 3 → 左边矮
  2 ≥ left_max(1) → left_max=2
  left=4, water=2

第9步：height[4]=1 vs height[7]=3
  1 < 3 → 左边矮
  1 < left_max(2) → water += 2-1 = 1
  left=5, water=3

第10步：height[5]=0 vs height[7]=3
  0 < 3 → 左边矮
  0 < left_max(2) → water += 2-0 = 2
  left=6, water=5

第11步：height[6]=1 vs height[7]=3
  1 < 3 → 左边矮
  1 < left_max(2) → water += 2-1 = 1
  left=7, water=6

left=7, right=7 → 相遇，结束！

总共接了6单位水 ✓
```

```
 2|  ■ ~ ~ ~ ■ ~ ■ ■ ~ ■
```

#### 代码实现

```python
def trap(height: list[int]) -> int:
    if not height:
        return 0

    left, right = 0, len(height) - 1
    left_max = right_max = 0
    water = 0

    while left < right:
        if height[left] < height[right]:
            # 左边矮，由左边决定水位
            if height[left] >= left_max:
                left_max = height[left]  # 更新左边最高
            else:
                water += left_max - height[left]  # 接水
            left += 1
        else:
            # 右边矮（或相等），由右边决定水位
            if height[right] >= right_max:
                right_max = height[right]  # 更新右边最高
            else:
                water += right_max - height[right]  # 接水
            right -= 1

    return water
```

#### 复杂度分析

- **时间复杂度**：O(n)，每个柱子处理一次
- **空间复杂度**：O(1)

#### 面试追问方向

**Q：双指针法的原理是什么？**
A：每个位置能接的水量 = min(左边最高, 右边最高) - 当前高度。双指针法确保每次处理的一侧，其"另一边"的最高值已经正确——因为如果左边矮，说明左边的水量由左边最高（已经知道）决定，右边更高不影响左边水位。

**Q：如果柱子宽度不同怎么办？**
A：需要乘上对应的宽度。接水量 = 高度差 × 宽度。

**Q：二维接雨水怎么做？**
A：使用最小堆（优先队列），从外往内推。这是LeetCode 407。

### 📝 第4题：分糖果

**题目**：Alice 有 `n` 枚糖，其中第 `i` 枚糖的类型为 `candyType[i]`。Alice 注意到她的体重正在增长，所以去医院拜访了一位医生。医生建议 Alice 要少摄入糖分，只吃掉她所有糖的 `n/2` 枚即可（n 是偶数）。Alice 非常喜欢吃糖，她想要在遵循医生建议的情况下，尽可能吃到**不同种类**的糖。给你一个长度为 `n` 的整数数组 `candyType`，返回 Alice 在仅吃掉 `n/2` 枚糖的情况下，可以吃到糖的**最多种类数**。

**难度**：⭐ 简单

**示例**：
```
输入：candyType = [1, 1, 2, 2, 3, 3]
输出：3
解释：Alice有6颗糖，医生让她吃3颗。有3种不同的糖（1,2,3），她可以每样吃一颗，吃到3种。

输入：candyType = [1, 1, 2, 3]
输出：2
解释：4颗糖，可以吃2颗。有3种不同的糖（1,2,3），但最多只能吃2颗，所以最多2种。

输入：candyType = [6, 6, 6, 6]
输出：1
解释：4颗糖，可以吃2颗。但只有1种糖，所以最多1种。
```

#### 解题思路

**核心**：答案 = min(不同种类的数量, n/2)

为什么？因为：
- 如果糖果种类很多（比如6种，只能吃3颗），最多吃3种 → min(6, 3) = 3
- 如果糖果种类很少（比如1种，能吃3颗），最多吃1种 → min(1, 3) = 1

#### 手把手推演

```
candyType = [1, 1, 2, 3]

n = 4, n/2 = 2（能吃2颗）
不同种类 = {1, 2, 3} = 3种

答案 = min(3, 2) = 2

也就是说，她最多吃到2种不同的糖。
（比如吃1和2，或1和3，或2和3）
```

```
candyType = [1, 1, 2, 2, 3, 3]

n = 6, n/2 = 3（能吃3颗）
不同种类 = {1, 2, 3} = 3种

答案 = min(3, 3) = 3

她能吃到全部3种糖！
```

#### 代码实现

```python
def distributeCandies(candyType: list[int]) -> int:
    # 计算不同种类的数量
    unique_types = len(set(candyType))
    # 最多能吃 n/2 颗
    max_eat = len(candyType) // 2
    # 答案取较小值
    return min(unique_types, max_eat)
```

3行代码！

#### 复杂度分析

- **时间复杂度**：O(n)，`set()` 操作需要遍历数组
- **空间复杂度**：O(n)，存储不同种类

#### 面试追问方向

**Q：如果 n 是奇数怎么办？**
A：医生一般建议 n/2（向下取整），实际上 `//` 就是向下取整。

**Q：如果每种糖果都有数量限制，不能随便选？**
A：那就不是简单题了，需要用贪心或者哈希表统计每种糖果数量。

### 📝 第5题：罗马数字转整数

**题目**：罗马数字包含以下七种字符：`I, V, X, L, C, D, M`。给定一个罗马数字，将其转换成整数。

| 字符 | 数值 |
|------|------|
| I | 1 |
| V | 5 |
| X | 10 |
| L | 50 |
| C | 100 |
| D | 500 |
| M | 1000 |

**特殊规则**：通常情况下，罗马数字中小的数字在大的数字的右边。但也存在特例：
- I 可以放在 V(5) 和 X(10) 的左边，表示 4 和 9
- X 可以放在 L(50) 和 C(100) 的左边，表示 40 和 90
- C 可以放在 D(500) 和 M(1000) 的左边，表示 400 和 900

**难度**：⭐ 简单

**示例**：
```
输入："III"，输出：3
输入："IV"，输出：4
输入："IX"，输出：9
输入："LVIII"，输出：58（L=50, V=5, III=3）
输入："MCMXCIV"，输出：1994（M=1000, CM=900, XC=90, IV=4）
```

#### 解题思路

**核心观察**：从左往右遍历，如果当前字符 < 下一个字符，说明是"特殊情况"（4或9等），需要**减**去当前值；否则**加**上当前值。

```
正常情况（左大右小）：M(1000) + C(100) + M(1000) → 不，C在M左边...等一下

正确理解：
"MCMXCIV"
M(1000) → C < M → 减C → 1000 - 100 = 900
等等不对... 应该是：
M(1000) → 下一个C(100) < M(1000) → 但C是下一个，要判断当前和下一个的关系

更清晰的说：遍历时看当前字符和下一个字符
- 如果当前 < 下一个 → 减当前值
- 否则 → 加当前值
```

细想：
```
M C M X C I V
M(1000) → 下一个C(100) < M(1000)? 100 < 1000 → 加M → total=1000
C(100) → 下一个M(1000) > C(100)? 100 < 1000 → 减C → total=900
M(1000) → 下一个X(10) < M(1000)? → 加M → total=1900
X(10) → 下一个C(100) > X(10)? → 减X → total=1890
C(100) → 下一个I(1) < C(100)? → 加C → total=1990
I(1) → 下一个V(5) > I(1)? → 减I → total=1989
V(5) → 最后一个 → 加V → total=1994 ✓
```

#### 代码实现

```python
def romanToInt(s: str) -> int:
    # 定义映射表
    roman_map = {
        'I': 1,
        'V': 5,
        'X': 10,
        'L': 50,
        'C': 100,
        'D': 500,
        'M': 1000
    }

    total = 0
    n = len(s)

    for i in range(n):
        value = roman_map[s[i]]

        # 如果不是最后一个字符，且当前值 < 下一个值
        if i < n - 1 and value < roman_map[s[i + 1]]:
            total -= value  # 减（特殊情况：4, 9, 40, 90 等）
        else:
            total += value  # 加（正常情况）

    return total
```

#### 手把手推演：MCMXCIV

```
s = "M C M X C I V"
n = 7

i=0: s[0]='M'=1000, 下一个C=100, 1000>100 → 加 total=1000
i=1: s[1]='C'=100, 下一个M=1000, 100<1000 → 减 total=900
i=2: s[2]='M'=1000, 下一个X=10, 1000>10 → 加 total=1900
i=3: s[3]='X'=10, 下一个C=100, 10<100 → 减 total=1890
i=4: s[4]='C'=100, 下一个I=1, 100>1 → 加 total=1990
i=5: s[5]='I'=1, 下一个V=5, 1<5 → 减 total=1989
i=6: s[6]='V'=5, 最后一个 → 加 total=1994

结果：1994 ✓
```

#### 图解

```
M     C     M     X     C     I     V
1000  100   1000  10    100   1     5
 ↓    ↓     ↓    ↓     ↓    ↓     ↓
+1000 -100 +1000 -10  +100  -1    +5
=1000 =900 =1900=1890 =1990=1989 =1994

决策：
M: 1000 > 100(C) → 加
C: 100 < 1000(M) → 减（这是CM=900的一部分）
M: 1000 > 10(X) → 加
X: 10 < 100(C) → 减（这是XC=90的一部分）
C: 100 > 1(I) → 加
I: 1 < 5(V) → 减（这是IV=4的一部分）
V: 5 → 最后一位 → 加
```

#### 复杂度分析

- **时间复杂度**：O(n)，一次遍历
- **空间复杂度**：O(1)，只用了固定的映射表

#### 面试追问方向

**Q：整数转罗马数字怎么做？（LeetCode 12）**
A：反过来，从大到小用不同的单位不断减：

```python
def intToRoman(num: int) -> str:
    values = [1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1]
    symbols = ["M", "CM", "D", "CD", "C", "XC", "L", "XL", "X", "IX", "V", "IV", "I"]

    result = []
    for i in range(len(values)):
        while num >= values[i]:
            num -= values[i]
            result.append(symbols[i])
    return "".join(result)
```

**Q：罗马数字有什么特殊规则？**
A：不能连续超过3个相同字符（如`IIII`不合法，应该是`IV`），不过题目保证输入合法的罗马数字。

**Q：大数（4000以上）怎么处理？**
A：传统罗马数字用上划线表示千倍数，但LeetCode和OD只处理到1~3999。

### 📝 第6题：拥有最多糖果的孩子

**题目**：给你一个数组 `candies` 和一个整数 `extraCandies`，其中 `candies[i]` 代表第 `i` 个孩子拥有的糖果数目。对每一个孩子，检查是否存在一种方案，将额外的 `extraCandies` 个糖果分给这个孩子后，这个孩子拥有的糖果数量是**最多的**（大于等于其他所有孩子）。返回一个布尔数组，表示每个孩子能否成为拥有最多糖果的孩子。

**难度**：⭐ 简单

**示例**：
```
输入：candies = [2, 3, 5, 1, 3], extraCandies = 3
输出：[True, True, True, False, True]
解释：
  孩子0：2+3=5 ≥ 最多(5) → True
  孩子1：3+3=6 ≥ 5 → True
  孩子2：5+3=8 ≥ 5 → True（本来就已经最多）
  孩子3：1+3=4 < 5 → False
  孩子4：3+3=6 ≥ 5 → True
```

#### 解题思路

**两步走**：
1. 找到当前糖果最多的数量：`max_candy = max(candies)`
2. 遍历每个孩子，判断：`candies[i] + extraCandies ≥ max_candy`

#### 代码实现

```python
def kidsWithCandies(candies: list[int], extraCandies: int) -> list[bool]:
    max_candy = max(candies)
    return [candy + extraCandies >= max_candy for candy in candies]
```

1行代码！（加变量声明的话就2行）

#### 手把手推演

```
candies = [2, 3, 5, 1, 3], extraCandies = 3

第1步：找最大值 max_candy = max([2,3,5,1,3]) = 5

第2步：逐项判断
  孩子0：2+3=5 ≥ 5 → True
  孩子1：3+3=6 ≥ 5 → True
  孩子2：5+3=8 ≥ 5 → True
  孩子3：1+3=4 < 5 → False
  孩子4：3+3=6 ≥ 5 → True

结果：[True, True, True, False, True]
```

#### 复杂度分析

- **时间复杂度**：O(n)，一次找最大值 + 一次遍历判断
- **空间复杂度**：O(n)，结果数组（如果只输出不求数组，可以逐个打印，空间O(1)）

#### 面试追问方向

**Q：如果有多个孩子同时得到额外糖果呢？**
A：题目只考虑给一个孩子额外糖果的情况。如果同时给多个孩子发，题目的判断逻辑不同。

**Q：如果额外糖果数很大（10^9）呢？**
A：不影响，`extraCandies` 只是整数，计算时不会溢出。

```python
# 1. 前缀积+后缀积
def productExceptSelf(nums):
    n = len(nums)
    ans = [1] * n
    left = 1
    for i in range(n):
        ans[i] = left
        left *= nums[i]
    right = 1
    for i in range(n-1, -1, -1):
        ans[i] *= right
        right *= nums[i]
    return ans

# 2. 加油站贪心
def canCompleteCircuit(gas, cost):
    if sum(gas) < sum(cost):
        return -1
    start, tank = 0, 0
    for i in range(len(gas)):
        tank += gas[i] - cost[i]
        if tank < 0:
            start = i + 1
            tank = 0
    return start

# 3. 接雨水双指针
def trap(height):
    l, r = 0, len(height) - 1
    l_max = r_max = water = 0
    while l < r:
        if height[l] < height[r]:
            if height[l] >= l_max:
                l_max = height[l]
            else:
                water += l_max - height[l]
            l += 1
        else:
            if height[r] >= r_max:
                r_max = height[r]
            else:
                water += r_max - height[r]
            r -= 1
    return water
```
