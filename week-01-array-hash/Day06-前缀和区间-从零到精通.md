## 📌 前言：从"累加"到"区间"的思维跃迁

想象你是一个超市收银员。顾客买了一车东西，你一个一个扫码计价。这时来了一个顾客说："我不想知道总价，我想知道第3件到第7件商品一共多少钱。"

最笨的办法：你重新扫一遍第3到第7件商品——这和每次重新计算区间有什么区别？

聪明的办法：你每扫一件商品就在小票上记录**累计金额**。第3件到第7件的金额 = 第7件的累计金额 - 第2件的累计金额。

这就是**前缀和**的核心思想——**预计算，快查询**。

## 🧱 第一章：一维前缀和——从"逐个累加"到"瞬间求和"

### 1.1 什么是前缀和？

**定义**：`prefix[i]` 表示数组前 i 个元素的和。

```
原数组:  [1, 2, 3, 4, 5]
前缀和:  [1, 3, 6, 10, 15]
         ↑  ↑   ↑   ↑   ↑
       prefix[0] prefix[1] prefix[2] prefix[3] prefix[4]
```

更常用的定义方式：`prefix[i]` 表示前 i 个元素的和（prefix[0] = 0）。

```
原数组:    [1,  2,  3,  4,  5]
前缀和:  [0, 1,  3,  6, 10, 15]
         ↑
       prefix[0]=0（前0个元素的和）

区间 [l, r] 的和 = prefix[r+1] - prefix[l]
区间 [1, 3] 的和 = nums[1]+nums[2]+nums[3] = 2+3+4 = 9
                 = prefix[4] - prefix[1] = 10 - 1 = 9 ✅
```

**生活类比**：你的银行流水账单。每个月月底都有"本月结余"这个数——这就是前缀和。想知道3月到7月的总收支，就用7月底的结余减去2月底的结余。

### 1.2 构建前缀和

```python
def build_prefix(nums):
    n = len(nums)
    prefix = [0] * (n + 1)  # prefix[0] = 0
    for i in range(n):
        prefix[i + 1] = prefix[i] + nums[i]
    return prefix

# 使用
nums = [1, 2, 3, 4, 5]
prefix = build_prefix(nums)
# 求区间 [l, r]（闭区间）的和
l, r = 1, 3
sum_lr = prefix[r + 1] - prefix[l]  # 2+3+4=9
```

**⚠️ 注意**：`prefix[i]` 是前 i 个元素的累加和，所以 `prefix[r+1] - prefix[l]` 对应的是下标 `l` 到 `r` 的区间。

### 1.3 什么时候用前缀和？

**场景**：需要**多次**查询不同区间的和。
- 如果只查一次，直接累加就够了
- 如果要查很多次（10^4次甚至更多），前缀和就非常划算

**代价**：O(n) 时间预处理，O(1) 时间每次查询。

## 🎯 第二章：简单前缀和实战（最高海拔 + 中心下标）

### 2.1 问题1：寻找最高海拔

> 自行车手从海拔0出发，给你一个gain数组，`gain[i]` 是点i到点i+1的海拔变化。求骑行过程中的最高海拔。

**示例**：
```
gain = [-5, 1, 5, 0, -7]
海拔变化序列：起点0 → -5 → -4 → 1 → 1 → -6
最高海拔：1
```

**这不就是求前缀和的最大值嘛！**

```python
def largestAltitude(gain: list[int]) -> int:
    current = 0      # 当前海拔（就是前缀和）
    max_alt = 0      # 记录最高海拔
    
    for g in gain:
        current += g     # 累加（构建前缀和）
        max_alt = max(max_alt, current)  # 更新最大值
    
    return max_alt
```

**算法推演**：
```
gain = [-5, 1, 5, 0, -7]
current=0, max_alt=0

第1步：g=-5, current=0+(-5)=-5, max_alt=max(0,-5)=0
第2步：g=1,  current=-5+1=-4, max_alt=max(0,-4)=0
第3步：g=5,  current=-4+5=1, max_alt=max(0,1)=1
第4步：g=0,  current=1+0=1, max_alt=max(1,1)=1
第5步：g=-7, current=1+(-7)=-6, max_alt=max(1,-6)=1

结果：1 ✅
```

### 2.2 问题2：寻找数组的中心下标

> 中心下标：左侧所有元素的和 == 右侧所有元素的和。如果存在，返回最左边的；不存在返 -1。

**示例**：
```
nums = [1, 7, 3, 6, 5, 6]
中心下标 = 3，因为：
  左侧和 = 1+7+3 = 11
  右侧和 = 5+6 = 11
```

**核心思路**：
1. 先计算总和 `total`
2. 从左到右遍历，维护 `left_sum`
3. 右侧和 = `total - left_sum - nums[i]`
4. 当 `left_sum == total - left_sum - nums[i]` 时，返回 i

**生活类比**：一根平衡木，你站在上面，左右两边重量相等才能保持平衡。你从左走到右，左边的东西越来越多，右边越来越少——找到平衡点。

**算法推演**：
```
nums = [1, 7, 3, 6, 5, 6]
total = 28
left_sum = 0

i=0, num=1:
  左侧和 = 0
  右侧和 = 28 - 0 - 1 = 27
  0 ≠ 27 → 不匹配
  left_sum = 1

i=1, num=7:
  左侧和 = 1
  右侧和 = 28 - 1 - 7 = 20
  1 ≠ 20 → 不匹配
  left_sum = 8

i=2, num=3:
  左侧和 = 8
  右侧和 = 28 - 8 - 3 = 17
  8 ≠ 17 → 不匹配
  left_sum = 11

i=3, num=6:
  左侧和 = 11
  右侧和 = 28 - 11 - 6 = 11
  11 == 11 → 返回 3 ✅
```

**代码**：
```python
def pivotIndex(nums: list[int]) -> int:
    total = sum(nums)
    left_sum = 0
    
    for i, num in enumerate(nums):
        # 右侧和 = 总和 - 左侧和 - 当前元素
        if left_sum == total - left_sum - num:
            return i
        left_sum += num
    
    return -1
```

### 2.3 边界情况处理

**情况1：中心下标在左端**
```
nums = [2, 1, -1]
total = 2
left_sum = 0

i=0, num=2:
  左侧和 = 0（左边没有元素）
  右侧和 = 2 - 0 - 2 = 0
  0 == 0 → 返回 0 ✅
```

**情况2：中心下标在右端**
```
nums = [1, 2, 3]
total = 6
left_sum = 0

i=0: 0 vs 5 → 不匹配
i=1: 1 vs 3 → 不匹配
i=2: 3 vs 0 → 不匹配
遍历结束，返回 -1 ❌

等等，看这个：
nums = [1, 2, 0]
total = 3
i=0: 0 vs 2 → 不匹配
i=1: 1 vs 0 → 不匹配
i=2: 3 vs 0 - 3 → 3 != -3 → 不匹配 ... 也不对

分析：对于 nums = [2, 1, -1]
i=2, num=-1:
  左侧和 = 2+1 = 3
  右侧和 = total - left_sum - num = 2 - 3 - (-1) = 0
  3 ≠ 0 → 不匹配
  结果 -1

而中心下标应该是 0（题目示例3），上面推演正确返回了 0 ✅
```

## 🧱 第三章：二维前缀和——从"一维"到"二维"的推广

### 3.1 一维到二维的类比

**一维区间和**：`sum[l:r] = prefix[r+1] - prefix[l]`

**二维子矩阵和**：`sum[(r1,c1):(r2,c2)] = prefix[r2+1][c2+1] - prefix[r1][c2+1] - prefix[r2+1][c1] + prefix[r1][c1]`

### 3.2 容斥原理：一张图理解

想象一个矩阵，`prefix[i][j]` 表示从 (0,0) 到 (i-1,j-1) 的子矩阵和。

```
矩阵：
  a  b  c
  d  e  f
  g  h  i

prefix 比原矩阵大一圈（多一行一列0）：
  0  0  0  0
  0  a  a+b  a+b+c
  0  a+d  a+b+d+e  a+b+c+d+e+f
  0  a+d+g  a+b+d+e+g+h  a+b+c+d+e+f+g+h+i
```

求子矩阵 (r1,c1) 到 (r2,c2) 的和：

```
prefix[r2+1][c2+1]  → 整个大矩形
- prefix[r1][c2+1]  → 减去上面部分
- prefix[r2+1][c1]  → 减去左边部分
+ prefix[r1][c1]    → 加回重复减去的左上角
```

**生活类比**：你要计算一亩地里某一小块区域的收成。你不必重新收一遍，而是用"整亩收成 - 上面部分 - 左边部分 + 左上角（被减了两次）"。

### 3.3 代码实现

```python
class NumMatrix:
    def __init__(self, matrix: list[list[int]]):
        m, n = len(matrix), len(matrix[0])
        # prefix[i][j] 表示 (0,0) 到 (i-1,j-1) 的子矩阵和
        self.prefix = [[0] * (n + 1) for _ in range(m + 1)]
        
        for i in range(m):
            row_sum = 0  # 当前行的累计和（优化用）
            for j in range(n):
                row_sum += matrix[i][j]
                # prefix[i+1][j+1] = 上面部分 + 左边部分 + 当前元素
                # 上面部分 = prefix[i][j+1]
                # 左边部分 = row_sum（排除上面部分的当前行累计）
                self.prefix[i + 1][j + 1] = self.prefix[i][j + 1] + row_sum
    
    def sumRegion(self, row1: int, col1: int, row2: int, col2: int) -> int:
        p = self.prefix
        return (p[row2 + 1][col2 + 1]
                - p[row1][col2 + 1]     # 减去上面
                - p[row2 + 1][col1]     # 减去左边
                + p[row1][col1])        # 加回左上角
```

### 3.4 逐行推演

以示例矩阵为例：
```
matrix = [
  [3, 0, 1, 4, 2],
  [5, 6, 3, 2, 1],
  [1, 2, 0, 1, 5],
  [4, 1, 0, 1, 7],
  [1, 0, 3, 0, 5]
]
```

构建 prefix：
```
prefix (6行6列)：
[0, 0,  0,  0,  0,  0]
[0, 3,  3,  4,  8, 10]
[0, 8, 14, 18, 24, 27]
[0, 9, 17, 21, 28, 36]
[0, 13,22, 26, 34, 49]
[0, 14,23, 30, 38, 58]
```

查询 `sumRegion(2, 1, 4, 3)`：
```

- p[row1][col2+1] = p[2][4] = 24
- p[row2+1][col1] = p[5][1] = 14
+ p[row1][col1]   = p[2][1] = 8

38 - 24 - 14 + 8 = 8 ✅
```

手动验证：子矩阵从 (2,1) 到 (4,3) 对应：
```
[2, 0, 1],
[1, 0, 1],
[0, 3, 0]
```
和 = 2+0+1+1+0+1+0+3+0 = 8 ✅

### 3.5 常见错误

**错误1：prefix 索引搞错**
```python
# ❌ 错误
def sumRegion(self, row1, col1, row2, col2):
    return self.prefix[row2][col2] - self.prefix[row1-1][col2] - ...

# ✅ 正确
def sumRegion(self, row1, col1, row2, col2):
    return (self.prefix[row2 + 1][col2 + 1]
            - self.prefix[row1][col2 + 1]
            - self.prefix[row2 + 1][col1]
            + self.prefix[row1][col1])
```

**错误2：忘记处理多行查询优化**
初始化时用双重循环 + 行累计和优化（O(m·n)），而不是三重循环。

## 🧱 第四章：区间合并——三步法解决所有重叠问题

### 4.1 什么是区间合并？

给定一组区间 `[[1,3],[2,6],[8,10],[15,18]]`，合并所有重叠的区间：
- [1,3] 和 [2,6] 重叠 → 合并为 [1,6]
- [8,10] 独立 → 保留 [8,10]
- [15,18] 独立 → 保留 [15,18]

结果：`[[1,6],[8,10],[15,18]]`

**生活类比**：你的日程安排。3点到5点有A会议，4点到6点有B会议——它们时间重叠，实际上你整个下午3点到6点都在开会（合并为3点到6点）。

### 4.2 区间合并三步法

```
第1步：排序（按区间左端点从小到大）
第2步：遍历，判断当前区间和上一个区间是否重叠
  重叠条件：当前区间的左端点 ≤ 上一个区间的右端点
第3步：如果不重叠 → 把当前区间加入结果
      如果重叠 → 更新上一个区间的右端点为较大的那个
```

**代码模板**：

```python
def merge(intervals: list[list[int]]) -> list[list[int]]:
    if not intervals:
        return []
    
    # 第1步：排序
    intervals.sort(key=lambda x: x[0])
    
    # 第2步：初始化结果集
    merged = [intervals[0]]
    
    # 第3步：遍历并合并
    for interval in intervals[1:]:
        # 当前区间的左端点 > 上一个区间的右端点 → 不重叠
        if interval[0] > merged[-1][1]:
            merged.append(interval)
        else:
            # 重叠 → 更新右端点
            merged[-1][1] = max(merged[-1][1], interval[1])
    
    return merged
```

### 4.3 算法推演

```
intervals = [[1,3],[2,6],[8,10],[15,18]]

第1步：排序（已经按左端点排好了）
  intervals = [[1,3],[2,6],[8,10],[15,18]]

第2步：初始化 merged = [[1,3]]

第3步：遍历
  当前区间 [2,6]：
    interval[0]=2 ≤ merged[-1][1]=3 → 重叠！
    更新 merged[-1][1] = max(3,6) = 6
    merged = [[1,6]]

  当前区间 [8,10]：
    interval[0]=8 > merged[-1][1]=6 → 不重叠！
    merged.append([8,10])
    merged = [[1,6],[8,10]]

  当前区间 [15,18]：
    interval[0]=15 > merged[-1][1]=10 → 不重叠！
    merged.append([15,18])
    merged = [[1,6],[8,10],[15,18]]

结果：[[1,6],[8,10],[15,18]] ✅
```

### 4.4 边界情况

**情况1：相邻区间（右端点 == 左端点）**
```
intervals = [[1,4],[4,5]]
interval[0]=4 ≤ merged[-1][1]=4 → 重叠！
更新右端点 = max(4,5) = 5
结果：[[1,5]] ✅（题目说边界相接也算重叠）
```

**情况2：一个大区间包含另一个**
```
intervals = [[1,10],[2,6]]
interval[0]=2 ≤ merged[-1][1]=10 → 重叠！
更新右端点 = max(10,6) = 10
结果：[[1,10]] ✅
```

**情况3：完全独立**
```
intervals = [[1,2],[3,4],[5,6]]
全部不重叠，结果不变：[[1,2],[3,4],[5,6]] ✅
```

## 🎯 第五章：区间合并变体——"插入区间"（三步法实际应用）

### 5.1 问题

> 给你一个已按左端点排序的无重叠区间列表，插入一个新区间，必要时合并重叠部分。

**示例**：
```
intervals = [[1,3],[6,9]], newInterval = [2,5]
结果：[[1,5],[6,9]]
```

### 5.2 核心思路：分三个阶段处理

```
阶段1：将新区间左侧的所有不重叠区间加入结果
阶段2：合并所有与新区间重叠的区间
阶段3：将新区间右侧的所有不重叠区间加入结果
```

**生活类比**：你要把一本新书插入到书架中已经排好序的书中间。先看左侧的书（比新书小的），再处理和新书有交叠的书，再看右侧的书（比新书大的）。

**代码模板**：

```python
def insert(intervals: list[list[int]], newInterval: list[int]) -> list[list[int]]:
    result = []
    i, n = 0, len(intervals)
    
    # 阶段1：不重叠且在左侧
    # 条件：当前区间的右端点 < 新区间的左端点
    while i < n and intervals[i][1] < newInterval[0]:
        result.append(intervals[i])
        i += 1
    
    # 阶段2：合并重叠区间
    # 条件：当前区间的左端点 <= 新区间的右端点
    while i < n and intervals[i][0] <= newInterval[1]:
        newInterval[0] = min(newInterval[0], intervals[i][0])
        newInterval[1] = max(newInterval[1], intervals[i][1])
        i += 1
    result.append(newInterval)
    
    # 阶段3：不重叠且在右侧
    while i < n:
        result.append(intervals[i])
        i += 1
    
    return result
```

### 5.3 算法推演

```
intervals = [[1,2],[3,5],[6,7],[8,10],[12,16]]
newInterval = [4,8]

阶段1（左侧）：
  检查 [1,2]：2 < 4 → 加入 result = [[1,2]], i=1
  检查 [3,5]：5 >= 4 → 不满足，进入阶段2

阶段2（合并）：
  当前 [3,5]：3 ≤ 8 → 合并 newInterval = [min(4,3), max(8,5)] = [3,8], i=2
  当前 [6,7]：6 ≤ 8 → 合并 newInterval = [min(3,6), max(8,7)] = [3,8], i=3
  当前 [8,10]：8 ≤ 8 → 合并 newInterval = [min(3,8), max(8,10)] = [3,10], i=4
  当前 [12,16]：12 > 10 → 不满足，退出
  result = [[1,2], [3,10]], i=4

阶段3（右侧）：
  加入 [12,16] → result = [[1,2], [3,10], [12,16]], i=5

结果：[[1,2],[3,10],[12,16]] ✅
```

### 5.4 边界情况

**情况1：新区间在最左侧**
```
intervals = [[3,5],[6,9]], newInterval = [1,2]
阶段1：3<1? 不，3≥1 → 直接进入阶段1（但3>2, 所以3<2? 不成立）
  检查 [3,5]：5<1? 不成立 → 阶段1不处理
阶段2：3≤2? 不成立 → 不合并
阶段3：加入所有 → result = [[1,2],[3,5],[6,9]]
```

等等，上面的推演不对。让我重新推演：

```
intervals = [[3,5],[6,9]], newInterval = [1,2]

阶段1：检查 [3,5]：5 < 1? 不成立 → 阶段1不处理，i=0

阶段2：检查 [3,5]：3 ≤ 2? 不成立 → 阶段2不处理

阶段3：加入 [3,5], [6,9]
结果中加入 newInterval[1,2]... 不对！

顺序应该是：阶段1后不满足就阶段2，阶段2后加newInterval，再阶段3。
但阶段2也不满足，newInterval应该在什么时候加？

重新看代码：阶段2的 while 循环不执行，但阶段2之后有 result.append(newInterval)！
所以 result = [[1,2]]，然后再加入 [3,5], [6,9]

结果：[[1,2],[3,5],[6,9]] ✅
```

**情况2：新区间在最右侧**
```

阶段1：1<6且2<6 → 加入[1,2], i=1
  3<6且5<6 → 加入[3,5], i=2
  结束阶段1

阶段2：i=2, n=2 → 不执行

阶段2后：result.append([6,8]) → [[1,2],[3,5],[6,8]]

阶段3：i=2, n=2 → 不执行
```

**情况3：新区间覆盖所有**
```

阶段1：2<1? 不成立 → 不处理

阶段2：[2,3]: 2≤10 → 合并为[1,10], i=1
  [5,6]: 5≤10 → 合并为[1,10], i=2
  result = [[1,10]]

阶段3：i=2, n=2 → 不执行

结果：[[1,10]] ✅
```

## 🎯 第六章：贪心射气球——"按右端点排序"的最优策略

### 6.1 问题

> 每个气球是一个区间 [xstart, xend]，一支箭在 x 处射出可以射中所有满足 xstart ≤ x ≤ xend 的气球。求射爆所有气球需要的最少箭数。

**示例**：
```
points = [[10,16],[2,8],[1,6],[7,12]]
最少箭数：2
  在 x=6 射 → 击中 [1,6],[2,8]
  在 x=11 射 → 击中 [7,12],[10,16]
```

### 6.2 核心思路：贪心（按右端点排序）

**贪心策略**：每次射箭都射在"最早结束"的气球的右端点上——这样能最大化这支箭覆盖的气球数量。

**为什么按右端点排序最优？**

**生活类比**：你是一个神箭手，面前有很多气球在水平线上飘过。你希望用最少的箭射爆所有气球。聪明的策略是：**先看哪个气球最快要飘走了（右端点最小），就在它飘走前射穿它**——这样还能顺便射中其他还没飘走的气球。

反过来说，如果你按左端点排序，可能会出现这种情况：
```
[1, 100]  [2, 3]  [4, 5]
按左端点排序：先射在1，能射中[1,100]但[2,3]和[4,5]也都能射中... 实际上只要3箭？
不对，在x=1射只能射中[1,100]，射不到[2,3]（因为x=1不在[2,3]内）
完全不是... [2,3]的xstart=2, xend=3，1 < 2，射不到。

按左端点排序射在[1,100]的右端点100 → 所有都射中，1箭。
但这只是巧合，因为[1,100]覆盖了所有。

看看标准反例：
[[1,2], [3,4], [5,6]]
按右端点排序 = [[1,2], [3,4], [5,6]]
箭1在x=2 → 射中[1,2]
箭2在x=4 → 射中[3,4]
箭3在x=6 → 射中[5,6]
3箭 ✓

如果按左端点排序射在第一个的左端点x=1：
只能射中[1,2]（因为[3,4]的左端点3>1，射不到）
同样需要3箭。

纯按左端点排序的贪心不一定最优，但有人证明对于射气球问题，按右端点排序是最优的（类似活动选择问题）。
```

### 6.3 算法推演

```
points = [[10,16],[2,8],[1,6],[7,12]]

第1步：按右端点排序
  sorted = [[1,6],[2,8],[7,12],[10,16]]

第2步：第一支箭射在第一个气球的右端点
  arrows = 1
  arrow_pos = 6

第3步：遍历剩余气球
  [2,8]：start=2 ≤ arrow_pos=6 → 这支箭能射中，跳过
  [7,12]：start=7 > arrow_pos=6 → 射不中！
    arrows = 2, arrow_pos = 12（新箭射在当前气球的右端点）
  [10,16]：start=10 ≤ arrow_pos=12 → 这支箭能射中，跳过

结果：2 ✅
```

### 6.4 代码实现

```python
def findMinArrowShots(points: list[list[int]]) -> int:
    if not points:
        return 0
    
    # 按右端点排序（关键！）
    points.sort(key=lambda x: x[1])
    
    arrows = 1
    arrow_pos = points[0][1]  # 第一支箭射在最早结束的气球右端点
    
    for start, end in points[1:]:
        # 如果当前气球的左端点 > 箭的位置 → 射不中
        if start > arrow_pos:
            arrows += 1
            arrow_pos = end  # 新箭射在当前气球右端点
    
    return arrows
```

### 6.5 为什么按右端点排序（详细证明）

想象两个气球：A = [a_start, a_end], B = [b_start, b_end]

情况1：如果不重叠（a_end < b_start），必然需要两支箭，排序不影响结果。

情况2：如果重叠，按右端点排序后，先处理右端点小的气球。

假设按右端点排序后，第一个气球的右端点是 R。在 R 处射箭能射中所有左端点 ≤ R 的气球。对于左端点 > R 的气球，这支箭射不中，需要新箭。

**关键点**：在 R 处射箭是最优的，因为：
- 射在 R 左边 → 可能射不中第一个气球的右半部分
- 射在 R 右边 → 第一个气球已经飞走了
- 射在 R 处 → 恰好射中最"紧迫"的气球，且最大化覆盖范围

### 6.6 常见错误

**错误1：按左端点排序**

```python
# ❌ 错误
points.sort(key=lambda x: x[0])  # 应该按右端点！

# ✅ 正确
points.sort(key=lambda x: x[1])  # 按右端点
```

**错误2：判断条件用了 >= 而不是 >**

```python
# ❌ 错误（气球边界重叠也可以被同一支箭射中）
if start >= arrow_pos:  # 应该用 >

# ✅ 正确
if start > arrow_pos:
```

等等，仔细看题目：`xstart ≤ x ≤ xend`。如果箭在 x=6，气球是 [2,6]，6 ≤ 6，应该能射中。但如果另一个气球是 [6,8]，start=6，arrow_pos=6，那这支箭也能射中 [6,8] 吗？6 ≤ 6 ≤ 8，能射中。

但如果按条件 `if start > arrow_pos`，6 > 6 不成立，所以这支箭被认为能射中 [6,8]——这是正确的！

那有没有边界情况？`start == arrow_pos` 时，箭在 x=arrow_pos 处，气球区间包含 x=arrow_pos → 能射中。所以 `>` 是对的。

**错误3：忘记处理空数组**
```python
if not points:
    return 0
```

### 2. 面试追问

**Q1：合并区间和插入区间的区别？**
A：合并区间针对**无序**区间，需要先排序；插入区间针对**有序**区间（已经排好序），可以用三阶段法更高效地处理。

**Q2：射气球问题为什么按右端点排序，而不是左端点？**
A：按右端点排序可以保证"在最早结束的气球的右端点射箭"是最优的。这是一个经典贪心策略，类似活动选择问题——在最早结束的活动的结束时间安排下一个活动。

**Q3：二维前缀和能否支持矩阵更新？**
A：如果矩阵同时需要更新和查询，应该用「二维树状数组（Fenwick Tree）」或「二维线段树」，时间复杂度 O(log m × log n)。

### 4. 常见错误大全

| 错误 | 原因 | 修正 |
|------|------|------|
| 合并区间忘记排序 | 认为输入已经有序 | 永远先排序 |
| 区间合并条件用错 | 用 `interval[0] < merged[-1][1]` 漏掉了边界相等 | 用 `>` 判断不重叠，否则合并 |
| 二维前缀和索引偏移 | 忘记 prefix 比原矩阵大一圈 | 牢记公式：`p[r2+1][c2+1] - p[r1][c2+1] - p[r2+1][c1] + p[r1][c1]` |
| 射气球按左端点排序 | 贪心策略错误 | 必须按右端点排序 |
| 插入区间忘记加 newInterval | 阶段2后忘记 append | 阶段2结束后一定有 result.append(newInterval) |

### 5. 前缀和 VS 滑动窗口对比

## 📚 今日总结

### 6个必须刻在脑子里的核心套路

1. **一维前缀和**：`prefix[i+1] = prefix[i] + nums[i]`，区间和 = `prefix[r+1] - prefix[l]`
2. **二维前缀和**：构建用四格公式，查询用容斥原理
3. **中心下标公式**：`left_sum == total - left_sum - nums[i]`
4. **区间合并三步法**：排序 → 判断（`interval[0] > merged[-1][1]`）→ 合并/追加
5. **插入区间三阶段**：左→合并→右
6. **贪心射气球**：按右端点排序，射在右端点

### 前缀和的思维总结

```
一维：  sum[l:r] = prefix[r+1] - prefix[l]
二维：  sum[(r1,c1):(r2,c2)] = p[r2+1][c2+1] - p[r1][c2+1] - p[r2+1][c1] + p[r1][c1]
```

**核心思想**：预计算 O(n)，查询 O(1)。典型的时间换空间策略。

### 一句话总结

> **前缀和让"求和"从 O(n) 变成 O(1)；区间合并三步法让所有区间问题有章可循；贪心射气球告诉你"在最早结束的地方出手"永远是明智的。**
