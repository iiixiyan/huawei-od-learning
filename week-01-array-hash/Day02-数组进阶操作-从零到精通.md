### ⚡ 套路一：三次翻转（轮转数组）

#### 🏠 生活例子：翻烤串 🥩

你有一串烤肉，要把最下面的几块翻到最上面：

```
原串：🍖🍗🥩🥓🍤 → 把最后2块翻到前面
目标：🥓🍤🍖🍗🥩
```

怎么做？不用重新串，只需要**三次翻转**！

```
第一次：整体翻转 → 🍤🥓🥩🥗🍖
第二次：翻转前2块 → 🥓🍤🥩🥗🍖
第三次：翻转后3块 → 🥓🍤🍖🥗🥩
搞定！
```

#### 核心思想

三次翻转的步骤：

```
原数组：    [1, 2, 3, 4, 5, 6, 7]   右移3步
               ↓
第1步：整体翻转 → [7, 6, 5, 4, 3, 2, 1]
第2步：翻转前k个 → [5, 6, 7, 4, 3, 2, 1]
第3步：翻转后n-k → [5, 6, 7, 1, 2, 3, 4]
               ↓
目标：      [5, 6, 7, 1, 2, 3, 4]   ✓
```

#### 代码模板

```python
def rotate(nums: list[int], k: int) -> None:
    n = len(nums)
    k %= n  # 防止k大于n

    # 辅助函数：反转 nums[left..right] 区间
    def reverse(l, r):
        while l < r:
            nums[l], nums[r] = nums[r], nums[l]
            l += 1
            r -= 1

    # 三次翻转
    reverse(0, n - 1)      # ① 整体翻转
    reverse(0, k - 1)      # ② 翻转前k个
    reverse(k, n - 1)      # ③ 翻转后n-k个
```

#### 为什么 k %= n ？

因为轮转 n 步等于没动——数组又回到原样。比如数组长度7，轮转8步等于轮转1步。

```
[1,2,3,4,5,6,7] 轮转7步 → [1,2,3,4,5,6,7] 原地不动
所以 k = k % n
```

#### 小白口诀 🎵

> **三次翻转三步走，整体、前段、后段扭。轮转问题不用愁，O(1)空间写爽透。**

### ⚡ 套路二：状态机（股票买卖I）

#### 🏠 生活例子：买卖玩具 🪀

你只有**一次**买卖机会。你知道未来每天的价格，怎么赚最多钱？

```
价格变化：7元 → 1元 → 5元 → 3元 → 6元 → 4元
          周一    周二    周三    周四    周五    周六
```

解法：只在最低价买入，最高价卖出（但要按时间顺序）。

这其实是个**状态机问题**——你只有两个状态：

```
状态0：未持有（还没买，或者卖完了）
状态1：持有中（已买入，还没卖）
      ↓
状态转移：
状态0 → 状态0（不买，继续观望）
状态0 → 状态1（买入，花掉钱）
状态1 → 状态1（不卖，继续持有）
状态1 → 状态0（卖出，赚差价）
```

因为只能买卖一次，状态机很简单——就是**找最低买入点**和**最大差价**。

#### 代码模板

```python
def maxProfit(prices: list[int]) -> int:
    min_price = float('inf')  # 历史最低价
    max_profit = 0            # 最大利润

    for price in prices:
        if price < min_price:
            min_price = price  # 更新最低买入价
        elif price - min_price > max_profit:
            max_profit = price - min_price  # 更新最大利润

    return max_profit
```

#### 小白口诀 🎵

> **最低价格记心里，每天价格比一比。差价更大就更新，一趟遍历没问题。**

### ⚡ 套路三：贪心策略（股票买卖II / 跳跃游戏）

#### 🏠 生活例子：爬山最优路径 ⛰️

你每次往前走两步，总能找到一条路。贪心就是：**每一步都选当前最好的选择，相信最终结果也是最好的。**

**贪心的三个要素**：
1. 局部最优 → 全局最优（需要证明）
2. 没有后顾之忧（不会因为贪心导致后面的路走不通）
3. 简单粗暴，不用回头看

#### 贪心 vs 动态规划

| 对比 | 贪心 | 动态规划 |
|------|------|----------|
| 决策 | 只看当前最优 | 考虑所有可能性 |
| 回头看 | 不用 | 需要 |
| 复杂度 | O(n) | 通常O(n²)或O(n×m) |
| 证明 | 需要数学证明 | 状态转移方程已验证 |

#### 代码模板（跳跃游戏）

```python
def canJump(nums: list[int]) -> bool:
    max_reach = 0  # 当前能跳到的最远位置
    for i in range(len(nums)):
        if i > max_reach:  # 到不了当前位置
            return False
        max_reach = max(max_reach, i + nums[i])  # 更新最远可达
        if max_reach >= len(nums) - 1:  # 已经能到终点
            return True
    return True
```

#### 小白口诀 🎵

> **贪心每次选最远，不断更新可达点。如果最远不到站，那就永远到不了。**

### 📝 第1题：轮转数组

**题目**：给定一个整数数组 `nums`，将数组中的元素**向右轮转** `k` 个位置。**原地**修改。

**难度**：⭐⭐ 中等

**示例**：
```
输入：nums = [1, 2, 3, 4, 5, 6, 7], k = 3
输出：[5, 6, 7, 1, 2, 3, 4]

输入：nums = [-1, -100, 3, 99], k = 2
输出：[3, 99, -1, -100]
```

#### 解题思路

三种方法，从简单到最优：

1. **暴力法**：每次右移1位，重复k次 → O(n×k)
2. **额外数组法**：开新数组，`new[(i+k)%n] = nums[i]` → O(n)空间
3. **三次翻转法**（最优）→ O(n)时间，O(1)空间

#### 三次翻转法的图解推演

```
nums = [1, 2, 3, 4, 5, 6, 7], k = 3

先处理 k: k = 3 % 7 = 3

第①步：整体翻转 [0, 6]
  [1, 2, 3, 4, 5, 6, 7]
   ↑                  ↑
   L                  R
   交换1和7 → [7, 2, 3, 4, 5, 6, 1]
       ↑            ↑
       L            R
   交换2和6 → [7, 6, 3, 4, 5, 2, 1]
          ↑      ↑
          L      R
   交换3和5 → [7, 6, 5, 4, 3, 2, 1]
             ↑↑
             LR（相遇，结束）
  结果：[7, 6, 5, 4, 3, 2, 1]

第②步：翻转前k个 [0, 2]（k=3，下标0~2）
  [7, 6, 5, 4, 3, 2, 1]
   ↑     ↑
   L     R
   交换7和5 → [5, 6, 7, 4, 3, 2, 1]
      ↑  ↑
      L  R（相遇，结束）
  结果：[5, 6, 7, 4, 3, 2, 1]

第③步：翻转后n-k个 [3, 6]
  [5, 6, 7, 4, 3, 2, 1]
            ↑         ↑
            L         R
   交换4和1 → [5, 6, 7, 1, 3, 2, 4]
               ↑      ↑
               L      R
   交换3和2 → [5, 6, 7, 1, 2, 3, 4]
                  ↑   ↑
                  L   R（相遇，结束）
```

#### 代码实现

```python
def rotate(nums: list[int], k: int) -> None:
    n = len(nums)
    k %= n

    def reverse(start: int, end: int) -> None:
        while start < end:
            nums[start], nums[end] = nums[end], nums[start]
            start += 1
            end -= 1

    reverse(0, n - 1)    # 整体翻转
    reverse(0, k - 1)    # 翻转前k个
    reverse(k, n - 1)    # 翻转后n-k个
```

#### 复杂度分析

- **时间复杂度**：O(n)，每个元素被翻转两次（整体一次+局部一次）
- **空间复杂度**：O(1)，原地修改

#### 面试追问方向

**Q：如果是左移怎么办？**
A：左移 k 位 = 右移 n-k 位。三步翻转也一样，只是第二步和第三步的划分不同。

**Q：三次翻转的原理是什么？**
A：数学原理：设数组为 AB，A 是前 n-k 个，B 是后 k 个。目标 BA。
- 整体翻转：A^R B^R
- 翻转前 k 个：B A^R
- 翻转后 n-k 个：B A ✓
（A^R 表示 A 的逆序）

### 📝 第2题：买卖股票的最佳时机I

**题目**：给定一个数组 `prices`，它的第 `i` 个元素 `prices[i]` 表示一支股票第 `i` 天的价格。你只能选择**某一天**买入这只股票，并选择在**未来的某一个不同的日子**卖出。返回你可以从这笔交易中获取的**最大利润**。如果不能获得任何利润，返回 `0`。

**限制**：只能买卖一次。

**难度**：⭐ 简单

**示例**：
```
输入：[7, 1, 5, 3, 6, 4]
输出：5
解释：在第2天（价格=1）买入，第5天（价格=6）卖出，利润=5。

输入：[7, 6, 4, 3, 1]
输出：0
解释：每天价格都在跌，不买不赚。
```

#### 解题思路

**状态机视角**：
- 状态0（未持有）：可以继续观望，或以今天的价格买入
- 状态1（持有中）：可以继续持有，或以今天的价格卖出

我们要最大化：卖出价格 - 买入价格（且卖出日 > 买入日）

**核心观察**：在第 i 天卖出能获得的最大利润 = prices[i] - 前 i 天的最低价格。

所以遍历一次，维护两个变量：
1. `min_price`：到当前为止的最低价格
2. `max_profit`：到目前为止能获得的最大利润

#### 手把手推演

```
prices = [7, 1, 5, 3, 6, 4]

初始：min_price = ∞, max_profit = 0

第0天：price=7
  7 < ∞ → min_price = 7
  profit = 7 - 7 = 0 → max_profit不变
  → min_price=7, max_profit=0

第1天：price=1
  1 < 7 → min_price = 1
  profit = 1 - 1 = 0 → max_profit不变
  → min_price=1, max_profit=0

第2天：price=5
  5 > 1 → min_price不变
  profit = 5 - 1 = 4 > 0 → max_profit = 4
  → min_price=1, max_profit=4

第3天：price=3
  3 > 1 → min_price不变
  profit = 3 - 1 = 2 < 4 → max_profit不变
  → min_price=1, max_profit=4

第4天：price=6
  6 > 1 → min_price不变
  profit = 6 - 1 = 5 > 4 → max_profit = 5
  → min_price=1, max_profit=5

第5天：price=4
  4 > 1 → min_price不变
  profit = 4 - 1 = 3 < 5 → max_profit不变
  → min_price=1, max_profit=5

结束：max_profit = 5
```

#### 图解

```
价格走势：
 7│   ●
 6│              ●
 5│        ●
 4│                    ●
 3│            ●
 2│
 1│        ●
 0└────────────────────→
    Day0 Day1 Day2 Day3 Day4 Day5

最低点：Day1的1
最高利润：Day4的6 - Day1的1 = 5

过程：
  Day0: min=7,  profit=0
  Day1: min=1,  profit=0      ← 更新最低价
  Day2: min=1,  profit=4      ← 利润=5-1=4
  Day3: min=1,  profit=4      ← 利润=3-1=2，不够大
  Day4: min=1,  profit=5      ← 利润=6-1=5，更新！
  Day5: min=1,  profit=5      ← 利润=4-1=3，不够大
```

#### 代码实现

```python
def maxProfit(prices: list[int]) -> int:
    min_price = float('inf')
    max_profit = 0

    for price in prices:
        if price < min_price:
            min_price = price  # 找到更低的买入价
        elif price - min_price > max_profit:
            max_profit = price - min_price  # 更新最大利润

    return max_profit
```

#### 复杂度分析

- **时间复杂度**：O(n)，一次遍历
- **空间复杂度**：O(1)

#### 面试追问方向

**Q：如果允许多次买卖呢？**（这是下一题）

**Q：如果允许卖空呢？**
A：卖空就是先卖后买，利润 = 买入价 - 卖出价。找最大跌幅即可。代码几乎一样，找最大差值（price - min_price 改为 max_price - price）。

**Q：如果要求最大利润的同时输出买入卖出的天数？**
A：记录更新 min_price 时的天数，以及更新 max_profit 时的卖出的天数。

### 📝 第3题：买卖股票的最佳时机II

**题目**：给你一个整数数组 `prices`，其中 `prices[i]` 表示某支股票第 `i` 天的价格。在每一天，你可以决定是否购买和/或出售股票。你在任何时候**最多只能持有一股**股票。你也可以**先购买，然后在同一天出售**。返回你能获得的**最大利润**。

**区别**：可以**无限次**买卖（但每次只能持有一股）。

**难度**：⭐⭐ 中等

**示例**：
```
输入：[7, 1, 5, 3, 6, 4]
输出：7
解释：
  第2天买入(1)，第3天卖出(5)，利润=4
  第4天买入(3)，第5天卖出(6)，利润=3
  总利润=7

输入：[1, 2, 3, 4, 5]
输出：4
解释：
  第1天买入(1)，第5天卖出(5)，利润=4
  （当然也可以每天买卖累积，但总和一样）

输入：[7, 6, 4, 3, 1]
输出：0
解释：一直跌，不买卖。
```

#### 解题思路

**贪心策略**：只要明天比今天贵，今天就买、明天就卖。

为什么这样是对的？
- 对于连续上涨的序列 `[1, 2, 3, 4, 5]`
  - 每天买卖：利润 = (2-1) + (3-2) + (4-3) + (5-4) = 4
  - 一次买卖：利润 = 5 - 1 = 4
  - 结果一样！
- 对于有涨有跌的 `[7, 1, 5, 3, 6, 4]`
  - 只涨的时候才买卖：1买5卖(赚4)，3买6卖(赚3) = 总利润7
  - 这就是所有上涨区间的总和

**核心思想**：把所有"正差价"加起来。

#### 手把手推演

```
prices = [7, 1, 5, 3, 6, 4]

遍历每一天，看明天是否比今天贵：

Day0→Day1: 7→1，跌了 → 不操作
Day1→Day2: 1→5，涨了4 → 利润+=4, profit=4
Day2→Day3: 5→3，跌了 → 不操作
Day3→Day4: 3→6，涨了3 → 利润+=3, profit=7
Day4→Day5: 6→4，跌了 → 不操作

总利润 = 4 + 3 = 7
```

#### 图解

```
价格走势：
 7│   ●
 6│              ●
 5│        ●     ↑ 涨3元
 4│    ↑        ●（跌）
 3│  涨4元    ●
 2│
 1│● （跌）
 0└────────────────────→
    D0  D1  D2  D3  D4  D5

只在箭头向上的时候买卖：
  1→5（涨4）
  3→6（涨3）
  总涨幅 = 7
```

#### 代码实现

```python
def maxProfit(prices: list[int]) -> int:
    profit = 0
    for i in range(1, len(prices)):
        if prices[i] > prices[i - 1]:  # 今天比昨天贵
            profit += prices[i] - prices[i - 1]  # 赚差价
    return profit
```

5行代码！这就是贪心的魅力。

#### 复杂度分析

- **时间复杂度**：O(n)，一次遍历
- **空间复杂度**：O(1)

#### 面试追问方向

**Q：为什么贪心在这里有效？**
A：因为买卖次数不限，每次交易没有成本。把每个上涨区间都捕捉到，总和就是最大利润。

**Q：如果交易有手续费怎么办？**
A：只有利润 > 手续费时才交易。需要状态机DP。状态0和状态1，转移时考虑手续费。

```python
def maxProfit(prices, fee):
    n = len(prices)
    hold = -prices[0]       # 第0天买入后的资产
    cash = 0                 # 第0天不做任何操作的资产

    for i in range(1, n):
        hold = max(hold, cash - prices[i])       # 买入 or 继续持有
        cash = max(cash, hold + prices[i] - fee) # 卖出 or 继续观望

    return cash
```

### 📝 第4题：跳跃游戏I

**题目**：给定一个非负整数数组 `nums`，你最初位于数组的**第一个下标**。数组中的每个元素代表你在该位置可以跳跃的**最大长度**。判断你是否能够到达**最后一个下标**。

**难度**：⭐⭐ 中等

**示例**：
```
输入：nums = [2, 3, 1, 1, 4]
输出：true
解释：可以先跳1步到位置1，再跳3步到终点。

输入：nums = [3, 2, 1, 0, 4]
输出：false
解释：无论怎么跳，都会卡在位置3（跳到0，跳不出去了）。
```

#### 解题思路

**贪心策略**：维护一个 `max_reach`（最远能到达的位置），遍历每个位置，不断更新它。

关键在于：
- 如果当前位置 `i` 超出了 `max_reach`，说明到不了这里 → 返回 false
- 否则，用 `i + nums[i]` 更新 `max_reach`（从当前位置起跳能到的最远位置）
- 如果 `max_reach` 已经 >= 最后位置 → 返回 true

**为什么贪心有效？** 因为跳得越远，选择越多。只要不跳过终点，多跳一步不会比少跳一步差。

#### 手把手推演

```
nums = [2, 3, 1, 1, 4]

初始：max_reach = 0

i=0: nums[0]=2
  i=0 ≤ max_reach=0 ✓
  max_reach = max(0, 0+2) = 2
  能到位置0~2
  [2, 3, 1, 1, 4]
   ↑
  max_reach→2

i=1: nums[1]=3
  i=1 ≤ max_reach=2 ✓
  max_reach = max(2, 1+3) = 4
  能到位置0~4（已经到终点了！）
  [2, 3, 1, 1, 4]
       ↑
  max_reach→4 ≥ len-1=4 → true!

返回 true
```

再看一个 false 的例子：

```
nums = [3, 2, 1, 0, 4]

初始：max_reach = 0

i=0: nums[0]=3
  max_reach = max(0, 0+3) = 3
  [3, 2, 1, 0, 4]
   ↑
  max_reach→3

i=1: nums[1]=2
  i=1 ≤ 3 ✓
  max_reach = max(3, 1+2) = 3  （没更新）

i=2: nums[2]=1
  i=2 ≤ 3 ✓
  max_reach = max(3, 2+1) = 3  （没更新）

i=3: nums[3]=0
  i=3 ≤ 3 ✓
  max_reach = max(3, 3+0) = 3  （没更新）

i=4: nums[4]=4
  i=4 > max_reach=3 ✗ → 到不了了！
  return false
```

#### 图解

情况一（可达）：
```
nums: [2, 3, 1, 1, 4]
下标:  0  1  2  3  4

可达范围变化：
  起始:  ■
  i=0:  ■■■■        (max_reach=2)
  i=1:  ■■■■■       (max_reach=4，覆盖终点 ✓)
```

情况二（不可达）：
```
nums: [3, 2, 1, 0, 4]
下标:  0  1  2  3  4

可达范围变化：
  起始:  ■
  i=0:  ■■■■        (max_reach=3)
  i=1:  ■■■■        (max_reach=3，没更新)
  i=2:  ■■■■        (max_reach=3，没更新)
  i=3:  ■■■■        (max_reach=3，卡在这)
  i=4:  到不了！✗   (i=4 > 3)
```

#### 代码实现

```python
def canJump(nums: list[int]) -> bool:
    max_reach = 0
    for i in range(len(nums)):
        if i > max_reach:  # 到不了当前位置
            return False
        max_reach = max(max_reach, i + nums[i])
        if max_reach >= len(nums) - 1:  # 已经到终点
            return True
    return True
```

#### 复杂度分析

- **时间复杂度**：O(n)，一次遍历
- **空间复杂度**：O(1)

#### 面试追问方向

**Q：如果改成最少需要跳跃几次？**（下一题：跳跃游戏II）

**Q：如果数组元素表示的是固定步数（不是最大步数）？**
A：那就是精确步数问题，用BFS或DP。

**Q：从后往前贪心怎么做？**
A：从终点往前找能到终点的最左边的位置，更新终点为那个位置，重复直到起点。

### 📝 第5题：跳跃游戏II

**题目**：给定一个长度为 `n` 的整数数组 `nums`。初始位置为 `nums[0]`。每个元素 `nums[i]` 表示从索引 `i` 向前跳转的最大长度。返回到达 `nums[n-1]` 的**最小跳跃次数**。生成的测试用例可以到达 `nums[n-1]`。

**难度**：⭐⭐⭐ 中等

**示例**：
```
输入：nums = [2, 3, 1, 1, 4]
输出：2
解释：跳1步到位置1，再跳3步到终点，最少2次。

输入：nums = [2, 3, 0, 1, 4]
输出：2
```

#### 解题思路

**贪心策略**：记录当前"一步"能跳到的最远边界。每次到达边界时，跳跃次数+1，更新边界为下一步能到的最远位置。

**核心思想**：在能跳到的范围内，找下一次能跳到的最远位置。就像玩跳棋，每次找最远的下落点。

**三个关键变量**：
- `max_reach`：在当前步数内能到达的最远位置
- `end`：当前步数的边界（到达这里就必须跳下一步）
- `steps`：已跳跃的次数

#### 手把手推演

```
nums = [2, 3, 1, 1, 4]

初始：max_reach=0, end=0, steps=0

i=0: nums[0]=2
  max_reach = max(0, 0+2) = 2
  i == end(=0) → steps=1, end=2
  → 第一步能到范围[0,2]

i=1: nums[1]=3
  max_reach = max(2, 1+3) = 4
  i(1) ≠ end(2)，还没到边界

i=2: nums[2]=1
  max_reach = max(4, 2+1) = 4  （没更新）
  i == end(=2) → steps=2, end=4
  end=4 ≥ len-1 → 到达！

结果：steps=2
```

#### 图解

```
nums: [2,  3,  1,  1,  4]
下标:  0   1   2   3   4

第一步（从0出发，可以跳到1或2）：
  范围：[0, 2]
  在这范围内，最远能跳到：i=1时3步→位置4
  所以第一步跳到位置2（最远可达边界）
  
  0 →→→→→→→→→→→→ 2
  |               |
  可到位置1      一步边界

第二步（在位置2，nums[2]=1，可跳到3）：
  但注意！上一步记录的下一步最远是位置4
  max_reach已经=4了
  所以从位置2到位置4只需要再跳一次
  
  2 →→ 3 →→→ 4  →→ 到达！
```

另一种直观理解：

```
起始：在0号位置（步数0）

第1跳能到：{1, 2}
  ↓
在第1跳的范围内（位置1和2）：
  位置1能跳到{2, 3, 4}
  位置2能跳到{3}
  所以第2跳能到：{2, 3, 4}
  ↓
位置4到达！共2跳。
```

#### 代码实现

```python
def jump(nums: list[int]) -> int:
    n = len(nums)
    if n == 1:
        return 0

    max_reach = 0  # 在当前步数内能到的最远位置
    end = 0        # 当前步数的边界
    steps = 0      # 跳跃次数

    for i in range(n - 1):  # 不需要遍历最后一个位置
        max_reach = max(max_reach, i + nums[i])

        if i == end:        # 到达当前步数的边界
            steps += 1      # 必须跳下一步
            end = max_reach # 更新边界为下次能到的最远位置
            if end >= n - 1:
                break

    return steps
```

#### 复杂度分析

- **时间复杂度**：O(n)，一次遍历
- **空间复杂度**：O(1)

#### 面试追问方向

**Q：和跳跃游戏I的区别？**
A：I只问能否到，这里问最少步数。I只用维护 max_reach，这里还需要 end 和 steps。

**Q：为什么循环到 n-2 而不是 n-1 ？**
A：因为不需要从最后一个位置起跳。如果 n == 1，已经到终点了，步数为0。

**Q：可以用BFS做吗？**
A：可以。把每个位置看作图中的节点，从 i 能跳到 i+1 ~ i+nums[i]，求最短路径。BFS的复杂度也是O(n)，但空间O(n)。贪心是O(1)空间。

### 📝 第6题：H指数

**题目**：给你一个整数数组 `citations`，其中 `citations[i]` 表示研究者的第 `i` 篇论文被引用的次数。计算并返回该研究者的 **h 指数**。

**h 指数**的定义：h 代表"高引用次数"（high citations），一名科研人员的 h 指数是指他（她）至少发表了 `h` 篇论文，并且**至少有 `h` 篇论文**被引用次数**大于等于 `h`**。其余 `n - h` 篇论文每篇被引用次数**不超过** `h`。

**难度**：⭐⭐⭐ 中等

**示例**：
```
输入：citations = [3, 0, 6, 1, 5]
输出：3
解释：有3篇论文至少被引用了3次（3, 6, 5），有2篇论文≤3次（0, 1）

输入：citations = [1, 3, 1]
输出：1
```

#### 解题思路

**核心观察**：将引用次数排序后，H指数就是满足 `citations[i] >= n - i` 的最大 `n-i`。

**推导**：
- 排序后：`[0, 1, 3, 5, 6]`
- 从大到小看：
  - 第1篇（6）：6 ≥ 1 → 至少有1篇引用≥1
  - 第2篇（5）：5 ≥ 2 → 至少有2篇引用≥2
  - 第3篇（3）：3 ≥ 3 → 至少有3篇引用≥3 ✓
  - 第4篇（1）：1 < 4 → 至少有4篇引用≥4 ✗
- 所以H=3

**另一种理解**：排序后从前往后遍历，找到第一个 `citations[i] >= n - i` 的位置，`n - i` 就是H指数。

#### 手把手推演

```
citations = [3, 0, 6, 1, 5]

第1步：排序
  [0, 1, 3, 5, 6]

第2步：从前往后遍历
  i=0: citations[0]=0, n-i=5, 0<5 → 还没有
  i=1: citations[1]=1, n-i=4, 1<4 → 还没有
  i=2: citations[2]=3, n-i=3, 3≥3 → 找到了！
  
  H = n - i = 5 - 2 = 3
```

#### 图解

```
排序后：
  引用次数:    0    1    3    5    6
  下标:        0    1    2    3    4
  需要≥次数:   5    4    3    2    1
  
  i=0: 0 ≥ 5? ✗
  i=1: 1 ≥ 4? ✗
  i=2: 3 ≥ 3? ✓  ← 找到！H = 5-2 = 3

直观理解：
  论文数n=5
  H=3 → 至少有3篇引用≥3，其余≤3
  [3, 6, 5] ≥ 3 → 3篇 ✓
  [0, 1] ≤ 3 → 2篇 ✓
```

#### 代码实现

```python
def hIndex(citations: list[int]) -> int:
    citations.sort()  # 升序排序
    n = len(citations)

    for i in range(n):
        h = n - i  # 从当前位置到末尾还有多少篇论文
        if citations[i] >= h:  # 满足"至少有h篇论文引用≥h"
            return h

    return 0  # 没有找到（理论上不会发生）
```

#### 复杂度分析

- **时间复杂度**：O(n log n)，排序占主导
- **空间复杂度**：O(1)（原地排序）或 O(n)（Python的Timsort需要额外空间）

#### 进阶解法：计数排序（O(n)时间）

如果引用次数范围有限（如OD中常见范围在0~1000），可以用计数排序：

```python
def hIndex(citations: list[int]) -> int:
    n = len(citations)
    papers = [0] * (n + 1)  # 统计每个引用次数有多少篇

    for c in citations:
        # 引用次数超过n的都记为n（因为H指数不会超过n）
        papers[min(c, n)] += 1

    # 从大到小累加论文数量
    cumulative = 0
    for h in range(n, -1, -1):
        cumulative += papers[h]
        if cumulative >= h:  # 至少有h篇论文引用次数≥h
            return h

    return 0
```

这样时间复杂度降到 O(n)！

#### 面试追问方向

**Q：H指数有什么实际意义？**
A：衡量科研人员的影响力。H指数越高说明高质量的论文越多。比如H=10表示至少有10篇论文被引用了10次以上。

**Q：H指数最大能到多少？**
A：不超过论文总数。所以引用次数超过n的都按n处理。

**Q：如果引用次数很大（10^9级），还能用计数排序吗？**
A：不能。这时只能用排序法O(n log n)。

### 模板速记

```python
# 1. 三次翻转模板
def rotate(nums, k):
    n = len(nums)
    k %= n
    def rev(l, r):
        while l < r:
            nums[l], nums[r] = nums[r], nums[l]
            l, r = l+1, r-1
    rev(0, n-1)
    rev(0, k-1)
    rev(k, n-1)

# 2. 股票I模板（一次买卖）
def maxProfit(prices):
    min_p = float('inf')
    max_pf = 0
    for p in prices:
        if p < min_p:
            min_p = p
        elif p - min_p > max_pf:
            max_pf = p - min_p
    return max_pf

# 3. 股票II模板（无限次买卖）
def maxProfit(prices):
    profit = 0
    for i in range(1, len(prices)):
        if prices[i] > prices[i-1]:
            profit += prices[i] - prices[i-1]
    return profit

# 4. 跳跃游戏I模板（能否到达）
def canJump(nums):
    max_r = 0
    for i in range(len(nums)):
        if i > max_r:
            return False
        max_r = max(max_r, i + nums[i])
    return True

# 5. 跳跃游戏II模板（最少步数）
def jump(nums):
    max_r = end = steps = 0
    for i in range(len(nums) - 1):
        max_r = max(max_r, i + nums[i])
        if i == end:
            steps += 1
            end = max_r
    return steps
```
